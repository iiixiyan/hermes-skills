# 权重优化技术

## 问题
大乐透10种分析方法各自有不同的预测能力，等权投票不是最优。
需要通过历史回测找到每个方法的最佳权重。

## 优化算法

### 两轮迭代搜索

```
Round 1: 逐个方法调优 (权重探索范围 0.3~3.0)
  heat=1.5 → 平均0.720个
  prime=0.3 → 平均0.740个
  Round 1 无改善 → 停止

最终权重: {'heat': 1.5, 'trend': 1.0, 'omission': 1.0, 
           'zone': 1.0, 'oe': 1.0, 'consec': 1.0,
           'pattern': 1.0, 'prime': 0.3, 'tail': 1.0}
```

### 优化指标
- **主要指标**: 平均命中数 (越高越好)
- **次要指标**: ≥2号占比 (差异<0.003时比较)

### 注意事项
1. 权重优化用的训练集（50期）与测试集（100期）必须分离，避免过拟合
2. 最优权重会随数据分布变化，建议每季度重新优化
3. prime=0.3说明质数分析在大乐透中效果为负，降低权重后整体提升

## 性能

_full_backtest 慢的原因：
- `analyze_omission()` 每次扫描全量 `self.data` 
- 在backtest循环中每期重建 `self.data = td[:i]`
- 优化方法：减少 `_silent` 模式下的冗余输出
