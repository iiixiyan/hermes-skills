# DLT 5注打乱重排工作流

> 2026-06-29 用户需求: "参考近五期区间进行打乱"

## 触发条件

生成5注推荐后、输出前执行。任何时候用户说"打乱"或"重新分配"时使用。

## 第一步：近5期区间分析

```python
zones = {'Z1(1-7)':range(1,8), 'Z2(8-14)':range(8,15), 
         'Z3(15-21)':range(15,22), 'Z4(22-28)':range(22,29), 
         'Z5(29-35)':range(29,36)}
```

统计近5期各区出现次数和空区次数：

| 指标 | 计算方式 | 用途 |
|:----|:---------|:-----|
| 空区次数 | 各区在5期内空白的期数 | 区间补缺优先级: 空越多优先级越高 |
| 各区遗漏号 | 近5期未出现的号码 | 优先选入5注 |
| 最近空区 | 哪一区在上一期空白 | 最强补缺信号(73%概率补号) |

## 第二步：识别25码池中的遗漏号

从当前25码池中标记哪些号是近5期遗漏号。

| 区 | 近5期遗漏号(不在pool则标注) |
|:---|:---------------------------|
| Z1 | 1,2,7 (2不在pool则跳过) |
| Z2 | 8,9,10,14 (8,9不在pool则跳过) |
| Z3 | 17,20 |
| Z4 | 23,25,27 |
| Z5 | 30,31,33,34,35 |

## 第三步：构造5注（每注5区各1号+遗漏号优先）

**规则：**
1. 每注必须涵盖Z1→Z2→Z3→Z4→Z5全部5个区间（各1号）
2. 各区内的选号顺序：近5期遗漏号 > 近期低频号 > 高频号
3. 如果某区遗漏号充足，优先分配遗漏号到不同注
4. 如可能，构建至少1注"全遗漏号注"（5个号全部是近5期遗漏的）

**构造示例：**

```
Pool: Z1=[1,3,4,5,7] Z2=[10,11,12,13,14] Z3=[15,18,19,20,21] Z4=[22,24,26,27,28] Z5=[29,31,32,33,34]
遗漏: Z1_miss=[1,7] Z2_miss=[10,14] Z3_miss=[20] Z4_miss=[27] Z5_miss=[31,33,34]

❶ 全遗漏注:  [1 (Z1), 10 (Z2), 20 (Z3), 27 (Z4), 33 (Z5)] ← 5区全遗漏号
❷ 区低号:    [3 (Z1), 11 (Z2), 15 (Z3), 22 (Z4), 29 (Z5)]
❸ 区中号:    [4 (Z1), 12 (Z2), 18 (Z3), 24 (Z4), 31 (Z5)] ← Z5遗漏
❹ 区高低混:  [5 (Z1), 13 (Z2), 19 (Z3), 26 (Z4), 34 (Z5)] ← Z5遗漏
❺ 区高号:    [7 (Z1), 14 (Z2), 21 (Z3), 28 (Z4), 32 (Z5)] ← Z1+Z2遗漏
```

## 第四步：验证

| 审查项 | 通过条件 |
|:------|:---------|
| 覆盖 | 25/35不变 |
| 五区间 | 每注5区间全覆 |
| 遗漏优先 | ❶注尽可能多遗漏号 |
| 后区 | 保留现有后区不变或不显著降低后区覆盖 |

## 后区保持规则

打乱只影响前区，后区保留原5对不变（已在上一轮优化中验证通过）。

## 与现有系统的集成

- 在 `5注推荐` 生成后、`对抗式审查压力测试` 之前执行
- 替代原来的"顺序排除分配"，使用"区间均衡+遗漏优先"分配

## v8.5.0 补充：大底重号控制

源自26073期复盘（8码大底含5重号，实际0重号导致前区0/5）

### 大底检查清单（输出前强制检查）

```
□ 重号数 ≤ 3？（8码中上期号最多3个，>3个则至少替换2个为新号）
□ 22或23在Z4？（中段号防盲区，不可全选24-28的高端号）
□ Z5含33或34？（高位盲区检查，不可遗漏）
□ 区间均衡？（5区间各≥1码？若某区缺，检查是否确实无合适号可选）
□ 如果上期≥2个重号？考虑0重号大底方案
```

### Z3空窗反弹机制

Z3(15-21)上期空窗时（26073期验证）：
1. 优先在Z3选15-17低位号（防补缺留在Z3内）
2. 同时选Z2高位(13,14)和Z4低位(22,23)做相邻区间覆盖
3. 避免同时选Z3+Z4远侧（如18+26），而应选15+22或17+23等靠近的相邻搭配
