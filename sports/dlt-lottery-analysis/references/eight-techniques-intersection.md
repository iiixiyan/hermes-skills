# 独孤八招×高分交集法 — 原理与验证

## 核心思想

通过两个完全独立的系统分别推荐号码，取交集得到"两方都认可"的共识号：

```
独立系统1: 独孤八招（每招选2号 → 16个推荐号）
独立系统2: 全维度评分（TOP12高分号）
          ↓
    交 集 = 16个技术号 ∩ 12个高分号 = 共识号
          ↓
    共识号 +20权重奖励 → 第1注优先选中
```

## 理论依据

两个系统各有盲区：
- **八招**是定性规则（间隔/冷热/连码），能发现评分系统忽视的号码（如深冷号）
- **评分**是定量统计（频率/遗漏/趋势），能修正八招的过度偏向
- **交集** = 两个盲区的补集 = 两方都认同的号码

## 选号流程（代码实现）

```python
# 见 _eight_techniques_vote() 和 get_5sets() 方法

# 第1步：八招投票(0-8票) + 高分排名奖励(0-10分)
weights[n] = votes.get(n, 0) * 1.0
           + max(0, 20 - rank) * 0.5  # 排名越前分越高
           + (20.0 if n in consensus else 0)  # 共识号重奖

# 第2步：第1注直接取权重TOP5（跳过区域限制，做连号控制）
s1 = [n for n, _ in sorted(weights.items(), key=lambda x: -x[1])[:5]]

# 第3步：第2-5注用pick_set（区域平衡+连号控制+不重叠）
s2 = pick_set(weights, exclude=s1)
...
```

## 独孤八招 — 每招详细选号规则

### 招❶：间隔状态选号
- **数据**: `ctx['oi'][n]['interval']` — 当前间隔超均值标准差的程度
- **范围**: 0.0-3.0
- **选号**: TOP8中超期号，取前2个

### 招❷：冷热转换选号
- **数据**: `ctx['transition'][n]` — 短窗(10期)vs长窗(50期)的变化
- **范围**: 0.0-3.5
- **选号**: TOP6升温号，取前2个

### 招❸：上期参考选号
- **数据**: `ctx['prev_ref'][n]` — 重号(3.0)+邻号(2.0)+隔号(1.2)+对称+钟摆
- **范围**: 0.0-5.0+
- **选号**: TOP10参考号，取前2个

### 招❹：多期参考选号
- **数据**: 近10期出现频率（`f10` Counter）
- **选号**: 出现≥2次的活跃号中取TOP2

### 招❺：相同尾数选号
- **数据**: 近30期尾数频率（`tail_freq`）
- **选号**: TOP4热尾中各尾数的评分最高号，取前2个

### 招❻：缩小选号范围
- **数据**: 近5期出现过的号码（`pool5`）
- **选号**: 按评分排序，取TOP2

### 招❼：连码重码
- **数据**: `ctx['tcp']`（TOP8连号对）+ 上期号码
- **选号**: 连号对成员 + 上期重号中取评分TOP2

### 招❽：冷码必不可少
- **数据**: `ctx['oi'][n]['omission']` — 遗漏期数
- **选号**: 遗漏≥15期中取遗漏最久的2个
