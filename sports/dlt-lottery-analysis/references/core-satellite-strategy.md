# 八招核心+卫星策略 (v5.5)

## 设计原理

独孤八招各自独立推荐号码 → 所有招数的推荐汇总 → 得票最高的3个号作为"核心"（恒星，每注出现）
→ 剩余推荐+补足共10个"卫星"（行星，各注不同） → 每注 = 3核心 + 2卫星

## 代码实现

`dlt_analyzer_pro.py` → `get_5sets()`:

## 优缺点

**优点：**
- 当核心准确时，所有注受益，爆发力强（26070期5注全≥2）
- 八招投票避免纯评分偏差
- 代码简洁，去除了复杂的权重系统

**缺点：**
- 覆盖仅13/35，核心不准时全崩
- ≥2率38.2%远低于非重叠66.6%
- 不适合不信任核心的场景
