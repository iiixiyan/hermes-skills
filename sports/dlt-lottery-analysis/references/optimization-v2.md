# DLT 5注 v2.0 优化记录（2026-06-23）

## 问题诊断

在回测26069期（实际开奖12-19-21-24-29 + 03-10）时发现：

| 问题 | 旧版(v1.x) | 影响 |
|:----|:----------|:-----|
| 后区覆盖极窄 | back_pool只取top6循环→实际仅3-4个不同号 | 后区命中0/5期 |
| 前区重叠严重 | 5注共用评分池→13-16/35个号 | 5注=实际2-3注 |
| 策略不分散 | 热号池仅10个号，4注重叠 | 浪费策略多样性 |

## 修复清单

### 修复1：后区双池策略

**文件：** `dlt_analyzer_pro.py` → `get_5sets()` 后区生成段

**旧逻辑：**
```python
back_pool = top6_by_score  # 只有6个候选
for i in range(5):
    b1 = back_pool[i % 6]
    b2 = back_pool[(i+2) % 6]
    # i=0和i=3结果相同！只生成3-4个唯一配对
```

**新逻辑：**
```python
all_12_back = 全部1-12按评分排序
top6 = all_12_back[:6]   # 高分号
bot6 = all_12_back[6:]   # 低冷号(被评分模型忽略的)
for i in range(5):
    if i < 3: 池 = top6   # 前3组从高分号取
    else: 池 = bot6       # 后2组从低冷号取
```
结果：后区覆盖 3-4 → **8-10** 个不同号

### 修复2：前区cross-set去重

**文件：** `dlt_analyzer_pro.py` → `pick_5()` + 策略调用

**旧逻辑：** 各策略独立pick_5，共用评分池
**新逻辑：** 后序策略exclude前序策略的已选号

```python
pick_5(src, exclude=set())  # 新增exclude参数
set1 = pick_5(sorted_scores)
set2 = pick_5(hot_sc, exclude=set(set1))
set3 = pick_5(cold_sc, exclude=set(set1+set2))
set4 = pick_5(pat_sc, exclude=set(set1+set2+set3))
```
结果：前区覆盖 13-16 → **25-27** 个不同号
