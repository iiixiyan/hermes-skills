# SSQ 5注Pro 遗漏模式分析（v2.4.0）

## 核心发现

**200期回测中，系统只中2红的期数占62%（124期）。** 分析这些"瓶颈期"后发现：

| 发现 | 数据 | 影响 |
|:---|:----|:----|
| 被遗漏号码中 **热号占62%** | 62%被遗漏号码是近30期TOP10热号 | 热号策略需要加强 |
| 被遗漏号码平均遗漏 | 24期（中等遗漏） | 不是极冷也不是极热 |
| 各区遗漏率 | 一区〜33% 二区〜33% 三区〜34% | 三区平均 |
| 各注命中（瓶颈期） | 热号追击0.84 > 高分0.76 > 分散0.75 > 梅法0.67 > 补漏0.57 | 覆盖补漏最弱 |

## 200期回测 + 三区覆盖修复（v2.5）

### 分段稳定性验证

| 段 | 平均 | ≥3红 | ≥4红 | 蓝球 |
|:-:|:---:|:----:|:----:|:---:|
| 200-151期 | 2.27 | 36% | 4% | 36% |
| 150-101期 | 2.22 | 28% | 4% | 30% |
| 100-51期 | 2.26 | 32% | 2% | 50% |
| 50-1期(最近) | 2.22 | 24% | **10%** | 36% |
| **200期整体** | **2.24** | **30%** | **5%** | **38%** |

各段平均2.22-2.27，波动极小 → 系统稳健，非过拟合。

### 三区覆盖修复

**发现：** 200期回测中，**58%的遗漏号码集中在三区(23-33)**（三区19个 vs 一区6个+二区8个）。

**最常遗漏号码TOP（中1-2红期）：**
| 号码 | 遗漏率 |
|:---:|:------:|
| 30号 | 71% |
| 29号 | 67% |
| 23号 | 67% |
| 19号 | 60% |
| 28号 | 53% |

**根因：** 5注策略都偏向中小号，大号覆盖不足。

**修复方式：**
1. 高分散度策略 — 三区(`high:[29,30,31,32,33]`)多取1个大号
2. 短期活跃策略 — 29-33号加基础分+2（即使未出现也获得初始权重）

**效果：** 三区遗漏从19个降至15个。[有待更大样本验证]

## 瓶颈分析方法（通用）

### 第一轮（v2.3）：短期活跃策略

**覆盖补漏(0.57个) → 短期活跃策略(近15期频率+近5期动量)**

```
score = freq_15 * 2 + (3 if in_momentum_5 else 0)
```

### 第二轮（v2.4）：尖峰检测

在短期活跃中加入**尖峰检测**（近5期频率 > 前10期频率 × 1.5 = 加速信号）：

```
score += 5 if (freq_5 > 0 and freq_5 > freq_10_prev * 1.5)
```

加速信号号可**突破分区上限限制**优先入选。这捕捉到那些"最近突然变热"的号码，比单纯频率统计更敏感。

## 效果

| 指标 | v2.2基线 | v2.3短期活跃 | **v2.4尖峰检测** | 
|:----|:-------:|:-----------:|:---------------:|
| 平均红球 | 2.21个 | 2.19个 | **2.22个 🏆** |
| ≥4红 | 4.0% | **5.0%** | **5.0%** |
| ≥3红 | 27.0% | **28.0%** | **28.0%** |
| 蓝球 | 39% | 38% | 38% |

## 瓶颈分析方法（通用）

对有类似瓶颈的系统（大乐透DLT等），通用分析流程：

1. 回测N期，按命中数分组（中0/1/2/3+红）
2. 分析低命中组的遗漏号码特征：
   - 近30期是否热号？ → 短期热号策略需加强
   - 遗漏值多大？ → 深冷/中冷/温号？
   - 哪个区间最多？ → 一/二/三区？
3. 找出N注中表现最弱的策略（指标：低命中期平均命中数）
4. 针对遗漏最多的号码类型，设计替代策略
5. 验证替代后是否改善了低命中期的分布
6. **迭代**：如果还有瓶颈，回到步骤2
