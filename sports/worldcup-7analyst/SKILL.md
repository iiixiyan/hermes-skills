# 世界杯7分析师预测体系（v2.1.0）

> 🏆 **核心理念**：7位独立分析师从不同维度分别打分，用数学模型将7个分差转为概率，再与市场赔率对撞，最终输出综合结论。
> 每个分析师是独立的"专家"，不知道其他分析师的打分——确保各维度信息不交叉污染。

---

## 一、触发条件

| 优先级 | 触发场景 |
|:------:|:---------|
| ⭐⭐⭐ | 用户要求预测世界杯比赛 |
| ⭐⭐⭐ | cron定时任务执行世界杯预测 |
| ⭐⭐ | 用户要求复盘世界杯预测 |

**边界规则：**
- ❌ 本技能仅用于世界杯正赛（小组赛+淘汰赛），不用于友谊赛/联赛/洲际杯赛
- ❌ 本技能不是 `football-prediction` 的替代品，是独立的7维方法论

## 二、完整管线总览（10步）

```
第1步：数据采集 — 全维度原始数据收集
    ↓
第2步：7位分析师独立评分 — 每人对两队打分，输出分差
    ↓
第3步：分析师权重配置 — 按比赛阶段/数据质量调整权重
    ↓
第4步：数学模型 — 7维分差 → 胜平负概率 + 比分概率
    ↓
第5步：市场赔率对撞 — 模型概率 vs 市场隐含概率
    ↓
第6步：Edge判定 — 是否存在有价值的方向偏差
    ↓
第7步：综合比分预测 — 融合模型+市场的最优比分推荐
    ↓
第8步：小组战略博弈评估（小组赛） / 淘汰赛路径评估（淘汰赛）
    ↓
第9步：串关/投注策略建议（按需）
    ↓
第10步：输出格式 — 标准化结论报告
```

### 3.4 关键数据处理说明

**雷速战绩数组 → 分析师②输入：**

**新浪百家平均 → 市场赔率对撞输入：**

```python
o1s = [float(o['o1New']) for o in odds if o.get('o1New')]
o2s = [float(o['o2New']) for o in odds if o.get('o2New')]
o3s = [float(o['o3New']) for o in odds if o.get('o3New')]
百家平均主胜 = sum(o1s) / len(o1s)   # 即赔均值
百家平均平局 = sum(o2s) / len(o2s)
百家平均客胜 = sum(o3s) / len(o3s)
```

## 四、第2步：7位分析师独立评分

### 4.1 评分标准

每位分析师使用 **0-100分制** 分别给A队和B队打分，然后计算 **分差 = A队分 - B队分**。

- 分差范围：-100 ~ +100
- 正值 = A队占优，负值 = B队占优
- 绝对值越大，该维度优势越明显

### 4.2 分析师详细评分规则

---

#### 分析师①：实力分析师（Strength Analyst）

**输入数据：** FIFA排名、总身价、大赛历史、阵容深度

**评分公式：**

```
实力得分_A = 50 + 20 × (FIFA_B_排名 - FIFA_A_排名) / 210   [FIFA排名分差]
          + 15 × sigmoid((身价_A - 身价_B) / 1亿)           [身价差异]
          + 15 × 大赛经验因子_A                               [世界杯/大赛历史]
          + 20 × 阵容深度因子_A                               [可用球员数量/质量]
```

其中：
- `sigmoid(x) = 1 / (1 + e^(-x))` — 将身价差异映射到0~1
- `大赛经验因子` = 上一届世界杯成绩/国家队大赛荣誉（0.3~1.0）
- `阵容深度因子` = 可替换球员质量评估（0.3~1.0）— 基于23人大名单中替补球员的联赛级别

**简化版（数据不足时）：**
```
实力得分_A = 50 + 30 × (FIFA_B - FIFA_A) / 210 + 20 × (身价_A - 身价_B) / (身价_A + 身价_B)
```

**输出：** `S_strength = 实力得分_A - 实力得分_B` （-100 ~ +100）

---

#### 分析师②：状态分析师（Form Analyst）

**输入数据：** 近10场战绩、进失球走势、半全场形态、比赛类型

**数据来源：**
```python
# 🥇 雷速 home_scores/away_scores（首选 — JSON直取，无浏览器）
#   长度为7的整数数组，每场已完成的比赛得分数
#   示例 [2,1,1,1,3,0,0] → 近7场进球数 2,1,1,1,3,0,0（第1个=最近）
#   注意：此数组只含本队得分，不含对手得分，也不含胜负状态
#   因此需要配合 59itou 战绩Tab来获取完整的对手比分和主客场信息
home_goals = match.home_scores  # 近7场本队进球数
away_goals = match.away_scores  # 近7场本队进球数

# 🥈 59itou战绩Tab（完整对阵明细 — 含对手名、比分、主客场、半全场）
#   当雷速数组缺乏对手信息时，需要来此处补全
```

**评分公式：**

```
状态得分_A = 近期战绩得分(50%) + 攻防数据得分(30%) + 走势动能得分(20%)
```

**近期战绩得分（40分基准）：**
```
近10场胜率 × 30 + 近5场胜率 × 10    [近5场权重更高]
胜率 = 胜场数 / 总场数
有加时赛算胜负，点球决胜算平局（FIFA统计规则）
```

**攻防数据得分（30分基准）：**
```
进攻效率 = 总进球 / 场次（近10场）
防守效率 = 总失球 / 场次（近10场）
攻防净差 = 进攻效率 - 防守效率
攻防得分 = 15 × tanh(攻防净差_A - 攻防净差_B)    [压制在±15]
```

**走势动能得分（20分基准）：**
```
连胜/连败修正：
· 近3场全胜 → +5
· 近3场全负 → -5
· 近5场赢盘率 > 60% → +3
· 近5场赢盘率 < 30% → -3
· 近2场进球数递增 → +2（进攻趋势向好）
· 近2场失球数递增 → -2（防守趋势恶化）
```

**输出：** `S_form = 状态得分_A - 状态得分_B` （-100 ~ +100）

---

#### 分析师③：战术分析师（Tactics Analyst）

**输入数据：** 常用阵型、教练风格、攻防策略、定位球效率、控球率

**评分维度（各占25分）：**

```
战术得分_A = 阵型适配度(25) + 教练博弈(25) + 攻防策略(25) + 定位球(25)
```

**阵型适配度（25分）：**
```
针对对方阵型的克制关系：
· 3中卫 vs 双前锋 → 3中卫方+5（人数优势）
· 4-3-3 vs 4-4-2 → 控球型+3（中场人数多）
· 5-4-1 vs 4-3-3 → 防守方+3（密集防守克制渗透）
· 相同阵型 → 0（无克制）
```

**教练博弈（25分）：**
```
· 教练世界杯经验：1届+3, 2届+6, 3届+10
· 教练对赛对手的熟悉度（来自同一大洲/联赛）：+3
· 教练临场调整历史（翻盘率 > 30%）：+5
```

**攻防策略（25分）：**
```
· 控球率 > 55% 且高效转化（射正率 > 35%）：+5
· 反击效率（反击进球占比 > 30%）：+3
· 高位逼抢强度（前场断球数/场 > 5）：+3
· 防守组织性（近5场失球来自反击的比例 < 20%）：+3
```

**定位球（25分）：**
```
· 角球进球率 > 5%：+5
· 任意球进球率 > 3%：+5
· 定位球防守（失球率 < 3%）：+5
· 有身高优势（平均身高 > 185cm）：+5
```

**输出：** `S_tactics = 战术得分_A - 战术得分_B` （-100 ~ +100）

---

#### 分析师④：关键球员分析师（Key Player Analyst）

**输入数据：** 核心球员名单、进球/助攻占比、关键球员状态、团队依赖性

**评分公式：**

```
关键球员得分_A = 进攻核心贡献(40) + 防守核心稳定性(30) + 板凳深度(30)
```

**进攻核心贡献（40分）：**
```
· 头号射手进球占比 > 40%：+10（依赖核心但也是风险——核心被锁=进攻瘫痪）
· 头号射手进球占比 20~40%：+15（健康分工）
· 头号射手进球占比 < 20%：+20（多点开花，最健康）
· 助攻王助攻占比 > 30%：+5
· 核心球员大赛经验（上届世界杯进球/助攻）：+10
```

**防守核心稳定性（30分）：**
```
· 门将稳定性（近10场扑救成功率 > 75%）：+10
· 主力中后卫搭档稳定性（搭档场次 > 15场）：+10
· 防线核心（队长/副队长在后防线）：+5
· 防守中场拦截能力（场均拦截 > 3次）：+5
```

**板凳深度（30分）：**
```
· 替补席有同一位置同等水平的球员：+10/人（最多+20）
· 替补球员近6场有进球/助攻：+5
· 23人名单中五大联赛球员占比 > 60%：+5
```

**输出：** `S_key_players = 关键球员得分_A - 关键球员得分_B` （-100 ~ +100）

---

#### 分析师⑤：伤停分析师（Injury/Suspension Analyst）

**输入数据：** 伤病名单、预计恢复时间、停赛球员、对攻防影响量化

**评分公式：**

```
伤停影响度_A = 核心球员缺阵惩罚 × 位置重要性因子 × 时间紧迫度因子
伤停得分_A = 50 - 伤停影响度_A  （50为基准，减分制）
```

**核心球员缺阵惩罚：**
```
· 头号射手缺阵 → -15
· 主力门将缺阵 → -12
· 队长/组织核心缺阵 → -10
· 主力中后卫缺阵 → -8
· 主力边后卫缺阵 → -5
· 主力中场缺阵 → -5
· 替补球员缺阵 → -2
```

**位置重要性因子：**
```
· 门将缺阵 → 1.0
· 后防核心缺阵 → 0.9
· 中场核心缺阵 → 0.8
· 前锋核心缺阵 → 0.7    [前锋位置可替代性更高]
```

**时间紧迫度因子：**
```
· 确认缺阵（已官宣排除出名单）→ 1.0
· 伤疑（赛前决定）→ 0.6
· 已恢复训练 → 0.2
```

**团队缺阵综合：**
```
伤停影响度_A = Σ(每位缺阵核心的惩罚 × 位置因子 × 时间因子)
```

**输出：** `S_injury = (50 - 伤停影响度_A) - (50 - 伤停影响度_B) = 伤停影响度_B - 伤停影响度_A` （-100 ~ +100）

---

#### 分析师⑥：战意/心理分析师（Motivation/Psychology Analyst）

**输入数据：** 出线形势、积分/净胜球、战意级别、历史交锋心理、大赛经验

**评分公式：**

```
战意得分_A = 出线形势得分(40) + 战意级别得分(30) + 心理因素得分(30)
```

**出线形势得分（40分）——小组赛专用：**
```
· 赢球即可出线 → +15
· 打平即可出线 → +10
· 大胜才能出线 → +5（背水一战，双刃剑）
```

## 六、第4步：数学模型 — 分差→概率转换

### 6.1 加权综合分差

```
D_weighted = Σ(w_i × S_i)   其中 i = 1~7
```
- `D_weighted` 范围：-100 ~ +100
- 正值 = A队占优，负值 = B队占优

### 6.2 胜平负概率转换（Logistic分布）

使用 **ELO派生公式**（基于Logistic分布），将综合分差转为概率：

```
P(A胜) = 1 / (1 + e^(-k × D_weighted / 100))
P(B胜) = 1 / (1 + e^(k × D_weighted / 100))
P(平局) = 1 - P(A胜) - P(B胜)
```

其中：
- `k = 1.6`（校准系数，基于历史数据拟合）
- 当 `|D_weighted| < 15` 时，平局概率额外上浮因子 `1.15`（均势局平局倾向）

**边界处理：**
```
D_weighted > 80 → P(A胜) 上限 85%，P(平局) 最低 5%
D_weighted < -80 → P(B胜) 上限 85%，P(平局) 最低 5%
```

### 6.3 预期进球（λ）计算

```
λ_A_base = 世界杯场均进球 × (50 + D_weighted) / 100
λ_B_base = 世界杯场均进球 × (50 - D_weighted) / 100
```

其中：**`世界杯场均进球 = 2.5`** （基于历史世界杯场均进球数，建议动态更新）

**λ修正因子（逐分析师贡献）：**

```
λ_A = λ_A_base × Π(R_i_修正因子)  其中 i = 1~7
λ_B = λ_B_base × Π(R_i_修正因子)
```

各分析师的λ修正因子：

| 分析师 | R修正因子规则 |
|:------|:-------------|
| ①实力 | S₁ > 30 → ×1.10（实力碾压导致多进球），S₁ < -30 → ×0.90 |
| ②状态 | S₂ > 20 且进攻趋势向上 → ×1.08，S₂ < -20 且防守崩溃 → ×1.12（对攻局） |
| ③战术 | 战术偏向进攻（4-3-3等）→ ×1.05，偏向防守（5-4-1等）→ ×0.92 |
| ④关键球员 | 进攻核心+头号射手高贡献 → ×1.10，防守型队伍 → ×0.95 |
| ⑤伤停 | 进攻核心伤缺 → ×0.80，防守核心伤缺 → 对方λ×1.10 |
| ⑥战意 | 必须大胜才能出线 → ×1.15，已出线轮换 → ×0.80 |
| ⑦环境 | 大雨/高温 → ×0.90（进球数减少），好天气 → ×1.00 |

### 6.4 比分概率分布（修正泊松）

使用 **Dixon-Coles τ修正泊松分布** 计算具体比分概率：

```
P(X=x, Y=y) = τ(x,y; ρ) × [e^(-λ_A) × λ_A^x / x!] × [e^(-λ_B) × λ_B^y / y!]
```

其中 `ρ = -0.10`（世界杯特定τ因子，低比分修正）

**比分概率排行：** 计算所有 0~5球 的组合（36种），按概率降序排列，取Top 6作为推荐比分池。

#### 比分发散度判定（比分范围的基础）

在选比分前，先计算该场的**比分发散度**，决定正常比分是往小球走还是大球走：

```python

# ⬆️ 发散因子（推高进球数）
if λ_A_final + λ_B_final > 2.8:   发散度得分 += 2   # 双方总预期进球高
if λ_A_final > 1.8:               发散度得分 += 1   # 强队λ高
if |D_weighted| > 40:             发散度得分 += 2   # 综合实力差距极大
if σ < 20 and D_weighted > 30:    发散度得分 += 1   # 分析师一致看好强队
if 让球 >= |-2|:                  发散度得分 += 1   # 深盘=预期大胜
if FIFA排名差 > 30:               发散度得分 += 2   # 🚨跨档对决！排名差>30即使首轮慢热也有碾压潜力（德国7-1库拉索模式）
if FIFA排名差 > 50:               发散度得分 += 1   # 极端跨档再加一级

# ⬇️ 收敛因子（压低进球数）
if 环境大雨/高温:                  发散度得分 -= 2
if 战意5/5且是淘汰赛:              发散度得分 -= 1   # 生死战保守
if 双方都防守型阵型(5-4-1等):     发散度得分 -= 1
if Edge强队负向且首轮:             发散度得分 -= 1   # 首轮+市场看衰

# 发散度等级
if 发散度得分 >= 3:  比分模式 = '大球模式'    # 正常比分含3-0/4-0
elif 发散度得分 >= 1: 比分模式 = '常规模式'    # 正常比分2-0/3-0级别

```

```python

    正常比分 = [3-0, 1-1]

    正常比分 = [3-0, 2-0]
```

**常规模式（发散度1~2）：正常比分选1个常规胜+1个防冷**

```python

elif 双轨判定 == '半背离':
    正常比分 = [2-0, 1-1]        # 半背离=强胜防平
elif 双轨判定 == '背离':
    正常比分 = [1-0, 1-1]        # 背离=小胜或冷平
elif 均势:
    正常比分 = [1-1, 0-0]
```

**小球模式（发散度≤0）：正常比分走小比分+防守**

```python
正常比分 = [1-0, 2-0]     # 双方进球受限，小比分决胜
if 双轨判定 == '背离':
    正常比分 = [1-1, 0-0]  # 背离更极端→双平
```

#### 爆冷比分双选规则（覆盖盘口反向+市场出货方向）

```
爆冷比分 = 双轨背离方向 + 弱队核心爆冷路径

选取原则（满足任意2条即触发强爆冷）：
· 双轨背离 → 爆冷方向 = 背离方向（市场指向谁，就爆谁）
· Edge弱队方向 > +5% → 爆冷选该方向赢球比分
· 分析师σ > 30且强队Edge负向 → ⚠️ 分析师打架+市场看衰=爆冷温床
· 让球 > |-2| 且升主胜极端(≥20家) → 深盘不防，冷门概率高
· 首轮比赛且强队让-1以上 → 爆冷必含1-1
· H2H历史弱队曾爆冷 → 参考历史爆冷路径

爆冷比分范围：
· 背离方赢1球 → 0-1 / 1-2 或 1-0 / 2-1（取决于背离方主客身份）
· 冷平 → 0-0 / 1-1（背离方只求不败）
· 大冷 → 0-2 / 1-3（背离方碾压式爆冷，罕见但需覆盖）

历史验证（2026世界杯D1）：
· 比利时1-1埃及 ✅ 双轨半背离 → 爆冷1-1覆盖
· 沙特1-1乌拉圭 ✅ 双轨背离(模型偏乌市场防平) → 爆冷1-1覆盖
· 西班牙0-0佛得角 ✅ 首轮+超低赔1.07 → 爆冷0-0
```

```
触发条件（满足任意1条）：

从修正泊松Top 6比分池中，按进球总和≥4筛选，取概率最高的2个。
```

## 十五、思维模型 — 第一性原理与对抗式审查

> 以下两种思维模型是贯穿世界杯7分析师预测体系最高层的方法论。每次分析前调用，作为认知框架的初始化。

### 15.1 概述

- **第一性原理** —— 每个分析师从原始数据出发独立打分，不因"市场共识"或"赔率方向"而调整自己的判断
- **对抗式审查** —— 7分析师共识形成后，主动挑战综合结论，发现群体思维盲区

### 15.2 第一性原理在7分析师体系中的体现

7分析师体系的设计本身就隐含了第一性原理——每个分析师从自己的维度独立打分，互不干扰。以下是各分析师的第一性校验：

| 分析师 | 第一性原理追问 | 常见偏差 |
|:------|:-------------|:---------|
| ①实力 | "FIFA排名差30位的真实含义是什么？" 排名差是结果指标，阵容换血/换帅可能使排名滞后 | 过度信任排名数字 |
| ②状态 | "近7场进球数组能否反映真实状态？" 对手强弱差异大时，弱队刷进球/强队踢强队进球少都需调整 | 未做对手强度调整 |
| ③战术 | "阵型克制的概率基础是什么？" 从532/433的对抗效率出发，而非"某个教练以克制冷门著称" | 用印象代替概率 |
| ④关键球员 | "没有这个球员的xG差值是多少？" 谁替代他？替代球员的效率差距有多大？ | 只关注缺阵不关注替代 |
| ⑤伤停 | "这个伤缺球员比赛中的实际触球位置/对攻防的量化影响是什么？" | 笼统减分不减到点上 |
| ⑥战意 | "积分情况下的客观战意是什么？" 而非"外界猜想的战意" | 用媒体报道替代积分计算 |
| ⑦环境 | "温度/海拔/裁判风格对不同球队战术的实际影响是什么？" | 模板化套用不做调整 |

**第一性原理校验清单（综合分差计算前强制执行）：**

```
□ 我的分析师评分是否基于原始数据（雷速home_scores/新浪百家平均/59itou阵容Tab），而不是基于印象或上场比赛结论？
□ σ标准差高的分析师，是否是因为我用了笼统模板而非逐场调整？
□ 双轨判定中，模型概率和市场概率不一致时——我回到基本面重新检查了吗？
```

### 15.3 对抗式审查在7分析师体系中的体现

7分析师体系已经有σ标准差作为一致性指标，但缺少显式的"挑战共识"步骤。对抗式审查在此体系中作为第7.5步——在综合分差计算后、最终比分输出前强制执行。

#### 15.3.1 σ标准差解读（已存在——对抗式审查的基础工具）

| σ范围 | 含义 | 对抗式动作 |
|:-----|:-----|:----------|
| σ < 15 | 高度一致 | ⚠️ 警惕群体思维！主动问：如果我们都错了，原因可能是什么？ |
| σ 15~25 | 正常分歧 | ✅ 正常可用，但检查是哪个分析师偏离方向 |
| σ > 25 | 显著分歧 | ⚠️ 不能简单取均值！找出分歧分析师，逐个审查其输入数据的可靠性 |
| σ > 30 | 严重分歧 | 🔴 市场赔率Edge优先级 > 模型综合分差 |

#### 15.3.2 对抗式审查四问（综合分差→最终输出前强制执行）

```
🔍 对抗式审查 —
1. 如果我必须给这场比赛的"冷门方"（模型预测的失败方）写一份3条有力的获胜理由，我能写出来吗？如果写不出来，说明我的分析有盲区。
2. 7分析师中，哪些维度的数据最不可靠？如果剔除最不可靠的2个分析师，综合结论会改变吗？
3. 市场赔率方向与模型方向相反（双轨背离）时，市场在"知道"什么我们不知道的事情？
4. 这场比赛最像近期哪场已完赛的比赛？那场比赛的真实结果验证还是推翻了类似的模型判断？
```

#### 15.3.3 赛后复盘中的对抗式审查

**挑战共识性解读：**

假设7分析师一致看好强队A（σ=12），但实际弱队B爆冷取胜。

- 你的分析："A队运气不好，B队防守反击执行完美"
- 🔍 对抗式挑战：
  - **"这是否是结果导向的合理化？** 赛后分析总能把任何结果都解释为合理的。关键在于：赛前分析师们一致看好的核心假设是什么？这个假设为什么在比赛中没有成立？
  - **"哪个分析师的偏差最大？** 是实力分析师高估了FIFA排名差？还是环境分析师忽略了高温对A队传控的影响？找出最偏离的分析师，问清楚原因
  - **"市场赔率赛前的信号是什么？** 如果市场已经在赛前示警（Edge负向），但分析师们无视了——这是模式级别的系统性偏差"

#### 15.3.4 对抗式审查在输出中的体现

```markdown
🛡️ 对抗式审查：
  σ = 14（高度一致 → ⚠️ 警惕群体思维）
  最脆弱假设：实力分基于FIFA排名，但A队近3场0胜从状态端看已走弱
  如果剔除状态分析师：D_weighted从+32降到+24，比分从3-0变为2-0
  市场Edge方向：与模型一致 → 双轨共振
```

### 15.4 在现有流程中的集成点

| 现有步骤 | 第一性原理集成 | 对抗式审查集成 |
|:---------|:-------------|:--------------|
| Step 2 分析师评分 | 每个分析师从原始数据出发，不参考其他分析师得分 | 评分完成后查σ，>25→标记为需人工审查 |
| Step 3 权重配置 | 权重调整必须基于数据质量而非个人偏好 | 高σ分析师权重应降低，而非取均值 |
| Step 4 数学模型 | 检查λ修正因子是否重复扣了同样的问题（如伤停同时在R和λ修正中减分） | 检查Edge方向与模型方向是否一致 |
| Step 5 市场赔率对撞 | 去抽水公式确认 | 双轨背离时启动对抗式四问 |
| Step 8a 小组战略 | 从积分/净胜球出发，而非"外界报道的战意" | "打平即出线"=死守？还是"打平出线"≈"输球也出线"？ |

### 15.5 铁律 — 两种思维模型的关系

```
第一性原理 → 构建（Build）：每个分析师从原始数据独立打分
    ↓
加权综合 → 构建完成：7维分差+市场对撞
    ↓
对抗式审查 → 检验（Test）：σ解读+四问+群体思维检查
    ↓
          → 风险清单足以改变方向 → 回到第一性重新检查可疑分析师的输入
          → 风险清单可控 → 输出最终预测
```

世界杯比赛容错率极低（每场只有一个结果），对抗式审查在这里的价值甚至高于联赛——因为世界杯的非确定性因素（大赛心理/首轮慢热/淘汰赛保守）远比联赛多，群体思维风险也更高。
