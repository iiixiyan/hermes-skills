# 北单每日复盘输出格式（联赛分区版）

> 用于每日复盘 cron job 的标准化输出模板。每次复盘输出必须严格按此格式排版。
> 由 2026-05-30（周六）37场复盘任务沉淀而来。

---

## 标准四段式结构

### 1️⃣ 总览

```
📋 **北单复盘报告 —— YYYY-MM-DD（星期X）**

**📊 总览：**
- 北单总场次：X场（全部采集自kt.59itou.com prize page + 详情页欧指/亚指）
- 共N个联赛/赛事
- 上盘(让胜)=X场(N%) | 走水(让平)=X场(N%) | 下盘(让负)=X场(N%)
```

### 2️⃣ 各联赛表现（联赛分组表）

**📈 各联赛表现：**

| 联赛 | 场次 | 让胜 | 让平 | 让负 | 上盘率 | 核心信号(+/-) |
|:----|:---:|:----:|:----:|:----:|:-----:|:-------------|
| [联赛名] | N | X | X | X | N% | [信号备注] |

**排序规则**：按场次数量降序排列。场次相同时按联赛名拼音排序。
**上盘率定义**：让胜场次/总场次。让平算走水不计入上下盘。
**核心信号**：只写该联赛最有特征的1-2个信号及准确率，用✅/⚠️/⛔标记。

### 3️⃣ 信号准确率矩阵（联赛×信号）

**🎯 信号准确率（按联赛）：**

| 信号 | 总体 | J1 | 芬超 | 瑞典超 | 瑞典甲 | K2 | 丹甲 | 其他联赛... |
|:----|:---:|:--:|:----:|:-----:|:-----:|:--:|:---:|:----------:|

**填写规则**：
- 每个格子填 "N/M"（正确数/触发数），括号外放总体百分比
- N/A表示该联赛该信号未触发
- 总体列放在最前，使用 16%时用 **⛔淘汰**颜色；60-74%用**🟡有效**；45-59%用**🟠边缘**；<30%用⛔标记

**必签信号列表**（按此顺序排列）：
1. 一致升主胜（胜升≥15且胜降≤3）
2. 一致降主胜（胜降≥15且胜升≤3）
3. 升主胜+降客胜
4. 百家主胜降赔（即赔<初赔）
5. 客胜升赔（即>初×1.05）
6. 主胜升赔（即>初×1.05）
7. 强升盘信号（升盘≥8且降盘≤2）
8. 强降盘信号（降盘≥8且升盘≤2）

**附加信号**（按需添加）：一致升客胜、一致降客胜、诱上降水不升盘、阻上升盘中高水、一致升主胜+降盘叠杀、一致降主胜+升盘阻上

### 4️⃣ 技能优化更新

```
**🛠 技能优化更新（bjdc-prediction vX.X.X）：**

| 更新项 | 内容 |
|:------|:-----|
| ✓ §16.2 信号追踪表已更新 | [具体更新的信号及其新准确率/状态] |
| ✓ 偏重判定N级优先级已修正 | [哪些联赛/条件被移动] |
| ✓ [联赛名]已[升级/降级] | [从xx到xx的变动说明] |
| ✓ 新增/修正[某项] | [详细说明] |
```

每次必须列出所有实际执行了的 SKILL.md 改动，使用 ✓ 标记完成项。

### 5️⃣ 关键发现/预警

```
**⚠️ 关键发现/预警：**

1. [联赛A]的XX信号准确率仅X%，需[降低权重/禁用/维持观察]
2. [联赛B]出现新的盘口规律：XXX
3. ...
```

**编号规则**：按严重程度排序，最重要的预警排第1位。每条预警后必须给出具体操作建议（降低权重/禁用/维持观察/升级）。

**预警必须锚定具体数据**，不能只说"信号不好"，要说"一致升主胜在J1联赛7次仅2正确(29%)"。

## 数据采集前置流程

复盘前必须完成的数据采集步骤：

1. **从BJDC prize page获取昨日比赛**
   - 导航至 `https://kt.59itou.com/danchang/prize/`
   - 提取昨日日期的所有 matchID（使用 grandparent DIV id 提取法）
   - 在同一 prize page 中解析比分、让球数（`(±N)` 格式）、联赛名
   - 检查最近2-3个批次（使用 `<p>` 元素切换 batch codes）

2. **从详情页获取欧指+亚指数据**
   - 使用 Playwright Python 批量采集（复用 browser/page）
   - 北单详情页URL：`lotteryId=45&lottery_style=dc`
   - 前缀从 `202` 开始尝试，内容<500字时循环 `["202","37","175","378","379","456"]`
   - 切换欧指Tab后用 `document.body.innerText` 提取
   - 提取指数变化（上升/降低家数）和百家平均（初赔/即赔）
   - 赛后24h内亚指Tab仍可用，超过3天仅欧指Tab可用

3. **信号分类与判定**
   - 赛后复盘时 JS click Tab 切换比 Hermes ref click 更稳定
   - 每场比赛约10-12秒（含欧指+亚指）
   - 单次 execute_code 300秒内可处理约25场比赛

## 每场比赛应记录的字段

| 字段 | 说明 | 来源 |
|:----|:-----|:-----|
| matchID | 比赛ID | prize page grandparent DIV id |
| 场次号 | 如"周六64" | prize page span.timetxt |
| 主队 | 中文队名 | prize page p.textr |
| 客队 | 中文队名 | prize page p.textl |
| 联赛/赛事 | 联赛名 | prize page p.liansai |
| 比分 | 如"1-1" | prize page b.bifen |
| 北单让球数 | 从 `(±N)` 格式提取或已知的赛前让球数 | 列表页赔率行或赛前采集数据 |
| 让球后结果 | 让胜/让平/让负（根据比分+让球数计算） | 计算所得 |
| 欧指变化 | 胜升/胜降/平升/平降/负升/负降各处家数 | 详情页欧指Tab |
| 亚指变化 | 升盘家数/降盘家数/主高水家数/主低水家数 | 详情页亚指Tab |
| 百家初赔 | [主胜, 平局, 客胜] 初赔 | 详情页欧指Tab |
| 百家即赔 | [主胜, 平局, 客胜] 即赔 | 详情页欧指Tab |

## 信号分类算法（Python）

```python
def classify_signals(ouzhi, yazhi):
    """Classify signals from extracted 欧指 and 亚指 data"""
    signals = []
    
    # 一致升主胜: 胜升≥15 and 胜降≤3
    if ouzhi.get('胜升', 0) >= 15 and ouzhi.get('胜降', 0) <= 3:
        signals.append('一致升主胜')
    
    # 一致降主胜: 胜降≥15 and 胜升≤3
    if ouzhi.get('胜降', 0) >= 15 and ouzhi.get('胜升', 0) <= 3:
        signals.append('一致降主胜')
    
    # 一致升客胜: 负升≥15 and 负降≤3
    if ouzhi.get('负升', 0) >= 15 and ouzhi.get('负降', 0) <= 3:
        signals.append('一致升客胜')
    
    # 一致降客胜: 负降≥15 and 负升≤3
    if ouzhi.get('负降', 0) >= 15 and ouzhi.get('负升', 0) <= 3:
        signals.append('一致降客胜')
    
    # 强升盘信号: 升盘≥8 and 降盘≤2
    if yazhi.get('升盘家数', 0) >= 8 and yazhi.get('降盘家数', 0) <= 2:
        signals.append('强升盘信号')
    
    # 强降盘信号: 降盘≥8 and 升盘≤2
    if yazhi.get('降盘家数', 0) >= 8 and yazhi.get('升盘家数', 0) <= 2:
        signals.append('强降盘信号')
    
    # 诱上型一致升主胜: 一致升主胜 + 降盘≥10
    if '一致升主胜' in signals and yazhi.get('降盘家数', 0) >= 10:
        signals.append('诱上降水不升盘')
    
    # 阻上型一致升主胜: 一致升主胜 + 降盘<5
    if '一致升主胜' in signals and yazhi.get('降盘家数', 0) < 5:
        signals.append('阻上升盘中高水')
    
    # 一致升主胜+降盘叠杀
    if '一致升主胜' in signals and yazhi.get('降盘家数', 0) >= 8:
        signals.append('一致升主胜+降盘叠杀')
    
    # 一致降主胜+升盘阻上
    if '一致降主胜' in signals and yazhi.get('升盘家数', 0) >= 8:
        signals.append('一致降主胜+升盘阻上')
    
    # 升主胜+降客胜
    if ouzhi.get('胜升', 0) > 5 and ouzhi.get('负降', 0) > 5:
        signals.append('升主胜+降客胜')
    
    # 百家主胜降赔 (即赔<初赔)
    if '初赔' in ouzhi and '即赔' in ouzhi:
        try:
            if float(ouzhi['即赔'][0]) < float(ouzhi['初赔'][0]):
                signals.append('百家主胜降赔')
        except: pass
    
    # 客胜升赔 (即>初×1.05)
    if '初赔' in ouzhi and '即赔' in ouzhi:
        try:
            if float(ouzhi['即赔'][2]) > float(ouzhi['初赔'][2]) * 1.05:
                signals.append('客胜升赔')
        except: pass
    
    # 主胜升赔 (即>初×1.05)
    if '初赔' in ouzhi and '即赔' in ouzhi:
        try:
            if float(ouzhi['即赔'][0]) > float(ouzhi['初赔'][0]) * 1.05:
                signals.append('主胜升赔')
        except: pass
    
    return signals
```

## 信号对应的方向判定

| 信号 | 指向方向 | 逻辑 |
|:----|:--------|:-----|
| 一致升主胜 | 让负(下盘) | 一致提高主胜赔率→打压主胜 |
| 一致降主胜 | 让胜(上盘) | 一致降低主胜赔率→看好主胜 |
| 一致升客胜 | 让胜(上盘) | 一致提高客胜赔率→主队有利 |
| 一致降客胜 | 让负(下盘) | 一致降低客胜赔率→客队有利 |
| 强升盘信号 | 让胜(上盘) | 多数公司升盘→看好主队 |
| 强降盘信号 | 让负(下盘) | 多数公司降盘→看衰主队 |
| 诱上降水不升盘 | 让负(下盘) | 降水不升盘=诱让球方 |
| 阻上升盘中高水 | 让胜(上盘) | 升盘中高水=阻让球方买盘 |
| 百家主胜降赔 | 让胜(上盘) | 百家赔率倾向主队 |
| 客胜升赔(即>初×1.05) | 让胜(上盘) | 客胜升赔→客队不被看好 |
| 主胜升赔(即>初×1.05) | 让负(下盘) | 主胜升赔→主队不被看好 |
| 升主胜+降客胜 | 走水/方向不明确 | 信号矛盾 |
