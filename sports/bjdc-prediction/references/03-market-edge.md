## 三-E、独立评级模型与Edge检测（v4.2 范式升级）

> **学术来源**：Constantinou (2022) *"Investigating the efficiency of the Asian handicap football betting market with ratings and Bayesian networks"* (arXiv:2003.09384) — 13个英超赛季数据证明AH市场不完全有效，独立评级模型+贝叶斯网络可系统性找到市场低效点。
> 补充：Clegg et al. (2026) 市场校准方法作为交叉验证使用。

### 核心范式转变

```
旧范式（v4.1）: 赔率 → 反推概率 → 决定方向  ❌ 盲信市场
新范式（v4.2）: 模型 → 独立概率 → 对比赔率找Edge → 决定方向  ✅ 模型先行
```

**为什么不能"北单三赔最低=市场看好方向"？**

| 问题 | 说明 |
|:----|:-----|
| 庄家margin不均 | 返还率65%，SP中已含margin，最低赔≠最高概率 |
| 热门-冷门偏差 | 市场系统性低估热门、高估冷门（favorite-longshot bias） |
| 庄家诱导 | 赔率本身包含诱导成分——降水可能是诱上而非真看好 |
| 市场非完全有效 | Constantinou论文证实AH市场存在可系统性利用的低效 |

### Edge检测三步法

#### 步骤1：模型先行——独立评估 P_model

不看赔率，纯基本面建模：

```
Glicko-2评级模型（简化版）:
  Rating分 = 近10场加权(近3×0.4+近6×0.3+赛季×0.3) + 主客场修正 + 伤停折损 + 战意系数

P_model 计算:
  1. 用评级分差值估算实力差距
  2. 映射到泊松λ（λ₁, λ₂）
  3. 应用双变量泊松（§三-F）+ DC τ修正（§三-G）
  4. 映射到让球盘 → P_model(让胜), P_model(让平), P_model(让负)
```

#### 步骤2：赔率后验——提取 P_market

从北单让球三赔提取隐含概率（**必须先去除庄家margin**）：

```
Raw: SP胜=2.16, SP平=3.57, SP负=3.87
总概率 = 1/2.16 + 1/3.57 + 1/3.87 = 0.463 + 0.280 + 0.258 = 1.001

P_market(让胜) = 0.463 / 1.001 = 46.3%
P_market(让平) = 0.280 / 1.001 = 28.0%
P_market(让负) = 0.258 / 1.001 = 25.8%
```

#### 步骤3：找Edge——差价驱动决策

```
Edge_i = P_model_i - P_market_i

决策:
  Edge > +5%  → 🟢 市场低估该方向 → 可出击
  Edge +3~5%  → 🟡 轻微低估 → 弱方向，需信号确认
  Edge ±3%    → ⚪ 模型与市场一致 → 无额外价值
  Edge -3~-5% → 🟠 市场略高估 → 谨慎
  Edge < -5%  → 🔴 市场高估该方向 → 回避

Edge + 欧指信号同向 → 强确认，权重×1.2
Edge + 欧指信号反向 → 降一档信心
所有方向Edge均不足±5% → 放弃该场
```

### Edge检测输出格式

每次分析时输出：
```
📊 Edge检测（v4.2）：
  P_model：让胜38% | 让平27% | 让负35%
  P_market：让胜32% | 让平30% | 让负38%
  Edge：  让胜+6%🟢 | 让平-3%⚪ | 让负-3%⚪
  方向：让胜（Edge+6%，市场低估）✅
  信号确认：[一致降主胜触发 → Edge与信号同向 → 权重×1.2]
```

### ⚠️ 实战优先级

C（Edge检测）> 信号交叉验证 > 双变量泊松修正 > DC τ修正
