# SkillOpt 训练循环 — 北单让球胜平负自适应优化

> 基于 SkillOpt 论文（arXiv:2605.23904）的工程化适配，把北单复盘→优化视为文本空间的训练循环。

## 核心映射

| SkillOpt 概念 | 北单场景实现 | 管控机制 |
|:-------------|:------------|:---------|
| **Rollout batch** | 早/晚场预测（每批~9场） | cron 11:00 / 22:00 |
| **Trajectory logging** | 预测让球方向→实际结果→偏差 | `review-insights.md` |
| **Minibatch reflection** | 按联赛分组复盘（每个联赛≥3场） | 联赛模板强制 |
| **Text learning rate Lₜ** | 每轮最多 **3 条**编辑 | 编辑预算计数器 |
| **Validation gate** | 同一数据集回测，必须严格涨分 | 验证报告 |
| **Rejected buffer** | 被拒绝的编辑方案 + 冷却期 | `16-rejected-edits-buffer.md` |
| **Slow/meta update** | 月度规律→写入联赛模板 | 月度复盘强制 |

## 一、训练循环流程

### Step 1: Rollout — 执行预测（每日）

- **早场 cron 11:00**：J1/K2/芬超/芬甲/瑞典甲下午场
- **晚场 cron 22:00**：挪超/瑞典超/意乙/罗甲/爱超等

### Step 2: 轨迹采集 — 获取实际结果

**数据源**：北单开奖结果页 `kt.59itou.com/danchang/prize/`

**采集规则**：
1. 北单 SP 值赛后放出，同竞足一样非同时开奖
2. 必须刷新页面多次直到所有场次出齐
3. 记录：预测让球方向 + 实际让球结果 + 偏差类型

### Step 3: Minibatch Reflection — 按信号类型分组

**分组维度**：
1. 按联赛（挪超一组、瑞典超一组、J1一组…）
2. 按盘口类型（让球-0.5、让球-1、平手…）
3. 按预测模式（模式一 vs 模式二 谁对谁错）

**必查偏差模式**：
- 某个联赛让球方被高估/低估 → λ系数调整
- 某种盘口类型的诱上/阻上误判率 → 盘口技术手册更新
- 某个联赛信号熔断触发频率异常 → 联赛阈值调整

### Step 4: 编辑提案（最多 3 条）

每轮最多 **3 条编辑**。格式：

```
编辑提案 N/N：
  目标：SKILL.md §X 或 references/xx.md §Y
  操作：add | delete | replace
  内容：[具体修改]
  预期收益：[量化]
```

### Step 5: 验证集门控

**门控条件**：
- 用候选 skill 在验证集上回测
- S_new > S_old 严格大于才接受（打平拒绝）

**验证集来源**：
- 日复盘：当天比赛数据
- 周复盘：过去7天有记录的所有比赛

### Step 6: 拒绝记账

被拒绝的编辑 → `16-rejected-edits-buffer.md`

### Step 7: 慢更新

月度复盘时将已验证通过的规则写入联赛模板（`references/05-league-templates.md`）

## 二、编辑预算

| 周期 | 上限 | 说明 |
|:----:|:----:|:------|
| 每日复盘 | 3 条 | 只修最显著的偏差 |
| 每周复盘 | 5 条 | 可做跨联赛迁移 |
| 月度复盘 | 8 条 | 结构性方法论调整 |

同一条规则连续被拒绝 2 次 → 冷却 2 周。

## 三、集成到现有复盘流程

**每次复盘前**：
1. 加载本文件（SkillOpt 训练循环）
2. 查询 `16-rejected-edits-buffer.md`（检查是否有冷却中的同名提案）
3. 加载 `references/10-review-methodology.md`（北单复盘方法论）
4. 加载 `references/11-review-insights.md`（历史规律）

**复盘时**：
1. 按联赛分组整理偏差矩阵
2. 找出跨场次的共性失败模式
3. 提案最多 3 条编辑
4. 验证集门控：S_old vs S_new

**复盘后**：
1. 通过 → 更新 SKILL.md/references + 版本号
2. 拒绝 → 移入 rejected buffer
