# 北单预测Cron Prompt 模板 (500-700字)

当为北单预测创建 cron job prompt 时，使用此模板。**核心原则：精简到 500-700 字**，避免加载过多参考文件导致 agent 卡死。

## 注意：北单分早场(11:00)和晚场(22:00)两个cron

北单比赛横跨 **12:55~次日06:00**，单个cron无法覆盖全部：
- **早场cron（11:00）**：覆盖12:55~19:00的比赛（J1/K2/芬超/芬甲/瑞典甲下午场）
- **晚场cron（22:00）**：覆盖22:00~06:00的比赛（挪超/瑞典超/芬超晚场/罗甲/意乙/法甲/爱超/冰岛超/智利甲/巴西乙）
- **SP预警cron（21:00）**：检测22:00场次SP漂移>15%，无异动不推送（静默模式）

两个cron共享北单列表页（kt.59itou.com/883/danchang/），但按开赛时间筛分后只输出对应批次。

## 完整模板（晚场22:00）

```
# 北单晚场预测（22:00）— 覆盖22:00~06:00凌晨场

获取北单全部比赛，**只输出开赛时间22:00~06:00的晚场**。早场由11:00 cron负责不重复。

每场单独输出完整分析，一场一发。输出格式：

〖北单-编号〗 [主] VS [客]（[联赛]）
📊 数据：排名/近6轮/伤停/三赔/H2H
🔍 模式一·八维 → 让胜/让平/让负
🔍 模式二·实力盘 → 方向
📌 因子：4b/12/13/联赛Edge
🔄 交叉：🟢一致 🟡有共性 🔴分歧
🎯 推荐：[A]/[B] ➡偏重[A] | ⚽X球 | ⭐X星

## 执行
1. 列表页获取全部比赛 → 按时间筛出晚场
2. 按编号逐场：详情页(lotteryId=45)→6Tab→分析→发送
3. 完场→总览表

## 禁止
❌ 合并/省略/篇幅原因/仅供参考/混用竞足/出比分
❌ 输出早场已踢完的比赛

## 参考
skill_view('59itou-data-fetch')
skill_view('bjdc-prediction')
```

## 早场模板差异（11:00）

```
# 北单早场预测（11:00）— 覆盖12:55~18:00

获取北单全部比赛，**只输出开赛时间12:55~19:00的早场**（J1/K2/芬超/芬甲/瑞典甲下午场）。

采集和分析流程与晚场相同。输出格式同上。

## 执行
1. 列表页 → 按时间筛出早场
2. 按编号逐场采集分析
3. 早场串关建议
```

## SP预警模板差异（21:00，静默模式）

```
# 临场SP异动预警（21:00）

检查今天所有22:00开赛的竞足+北单比赛的最新SP值，对比初始SP。
只推送SP变动>15%的比赛。

## 输出规则
- 无异动 → 不发消息（静默）
- 有异动 → ⚠️ SP异动预警：场次+原SP+现SP+变动幅度+建议

## 参考
skill_view('59itou-data-fetch')
```

## 设计原则

| 原则 | 说明 |
|:----|:-----|
| 500-700字 | 超过1000字+多个参考文件=agent易卡死 |
| 格式模板放在最前面 | cron agent先看到输出要求再执行 |
| 参考文件≤2个关键skill | 主skill + data-fetch skill即可 |
| 禁止列表精简 | 5-8条，一条一行 |
| 执行步骤3步以内 | 列表→逐场→总览 |
| SP预警静默输出 | 无异动不产生任何推送，节省token |
