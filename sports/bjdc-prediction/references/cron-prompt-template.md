# 北单预测Cron Prompt 模板 (500-700字)

当为北单预测创建 cron job prompt 时，使用此模板。**核心原则：精简到 500-700 字**，避免加载过多参考文件导致 agent 卡死。

## 完整模板

```
# 北单每日预测（22:00）— 逐场+双模

获取全部北单比赛，**每场单独输出完整分析**，一场一发。

## 输出格式
```
〖北单-编号〗 [主] VS [客]（[联赛]）

📊 数据：排名/近6轮/伤停/三赔/H2H

🔍 模式一·八维：联赛势位|主客|状态|交战|相似对手|初盘|走势|交叉
→ 让胜/让平/让负 | ⭐X星(X/8维)

🔍 模式二·实力盘：基准盘→实盘→偏差→水位→意图→指向

📌 因子：4b/12/13/联赛Edge

🔄 交叉：🟢一致 🟡有共性 🔴分歧

🎯 推荐：[A]/[B] ➡偏重[A] | ⚽X球/X球 | ⭐X星
```

## 执行
1. 列表：kt.59itou.com/883/danchang/ → matchID+三赔
2. 按编号逐场：详情页(lotteryId=45)→6Tab→分析→发送
3. 完场→总览表

## 禁止
❌ 合并/省略/跳过λ/篇幅原因/仅供参考/混用竞足/出比分

## 参考
skill_view('59itou-data-fetch')
skill_view('bjdc-prediction') → refs: 09预测法/04因子/05联赛/06诱阻/01泊松
```

## 设计原则

| 原则 | 说明 |
|:----|:-----|
| 500-700字 | 超过1000字+多个参考文件=agent易卡死 |
| 格式模板放在最前面 | cron agent先看到输出要求再执行 |
| 参考文件≤2个关键skill | 主skill + data-fetch skill即可，子引用在SKILL.md里加载 |
| 禁止列表精简 | 5-8条，一条一行，不用展开 |
| 执行步骤3步以内 | 列表→逐场→总览，避免复杂流程描述 |
