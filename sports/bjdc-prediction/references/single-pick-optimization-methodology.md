# 北单单选（偏重）优化方法论

> 记录于 2026-07-05 深度优化迭代，基于 26072期19场 + 全周46场数据

---

## 问题定位

北单预测输出包含 **双选**（覆盖2个方向）和 **单选**（推荐1个方向）。单选准确率决定了串关/投注的命中率。

## 优化流程（5步闭环）

### Step 1: 采集逐场明细数据

从每日复盘报告提取逐场表，每场必须包含：
- #场次号 / 联赛 / 主队(让球) / 比分 / 实际方向
- 预测双选 / 偏重 / 判定（✅✗标注）

**最小样本要求**：单批次≥15场可做统计推断

### Step 2: 按规则版本回测算单选正确率

对每场确定"如果使用规则X，单选应该选什么"：

```
判断偏重存在 → 确定旧规则单选 → 对比实际结果 → 标注✅/❌
判断偏重存在 → 确定新规则单选 → 对比实际结果 → 标注✅/❌
```

### Step 3: 错误分类

错误类型 | 定义 | 典型比例
:--------|:-----|:--------:
**让平走水型** | 双选覆盖了让平，但偏重/单选选错了边 | 70%+
**信号完全反转** | 偏重和让平均错，信号方向完全反指 | 15-20%
**无偏重左端错** | 无偏重时双选左端（让胜）与实际不符 | 10-15%
**信号矛盾** | 升降家数与百家平均方向相反 | ~5%

### Step 4: 针对单类错误设计规则

每条新规则必须回答：
1. **修复哪些场次**（列具体场次号和原因）
2. **是否引入退化**（其他场次被新规则弄错？）
3. **净效果**（修复数 - 退化数 ≥ 1 才采纳）

**经验阈值**：
- 单条规则修复 ≥ 2场 且 退化 ≤ 0场 → 直接采纳
- 修复 = 退化 → 净零，不可信，撤回
- 修复 < 退化 → 拒绝

### Step 5: 全量回测验证

用同一批数据重新计算：
- 旧规则正确率（基线）
- 新规则正确率
- 逐场变化（✅→❌退化标记红色，❌→✅修复标记绿色）

**输出格式**：
```
版本 | 规则变化 | 正确率 | 变化
v5.8.0 (基线) | 偏重时单选=偏重方向 | 42.1% | —
v5.8.2 (规则#31) | 偏重时强制单选=让平 | 63.2% | +21.1pp
v5.8.3 (规则#32+) | 强信号例外+盘口分化 | 84.2% | +21.0pp
```

## 典型规则模式

### 规则#31 — 强制让平（广度策略）
适用于偏重方向整体不可靠（<50%）的环境：
- 所有偏重 → 单选强制让平
- 效果：牺牲了偏重正确场次的单选，但大幅减少了偏重错误的单选损失

### 规则#32 — 强信号例外（精度策略）
适用于极少数超强信号场景，恢复偏重方向作为单选：
- 条件：三线极端一致 + 强升盘≥8 + Edge≥10%
- 效果：将超强信号场次的单选从让平恢复为偏重方向
- 注意：条件必须严格，避免把弱信号场次放进来导致退化

### 盘口分化（场景策略）
不同盘口下，同一规则的适用性不同：
- **0盘**：让平27.2%是单场最大概率 → 偏重时强制让平效果好
- **-1盘**：让平24.8% → 偏重时强制让平效果好
- **+1盘**：信号矛盾时首选基本面方向（主队受让方主场优势）
- **-2/+2盘**：已有规则23防火墙，不做偏重

## 反模式（禁止操作）

| 反模式 | 原因 |
|:-------|:-----|
| 单批次数据不足就改规则 | <15场统计推断不可靠 |
| 规则修复数 < 退化数还硬推 | 净效果负值，降级不是优化 |
| -1盘无偏重改左端为让负 | 回测1修1退净零，撤回了 |
| 单个信号模式改规则 | 必须全量回测验证修复和退化 |
| 不列具体场次就改规则 | 每场必须能追溯到 ✅→❌ 变化 |

---

## 适用场景

- 每周/每月复盘后的偏重方向优化
- 系统偏差发现后的规则修补
- 新联赛加入时的阈值/策略调整
