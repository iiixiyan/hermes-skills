## 二、核心概念与铁纪

### 数据源规范

| 维度 | 来源 |
|:----|:-----|
| **北单让球三赔**（最核心） | 列表页 `kt.59itou.com/883/danchang/` → DIV.id = matchID |
| **详情页欧指+亚指**（交叉验证） | `lotteryId=45&lottery_style=dc` → 5Tab提取 |
| **必发交易量/冷热指数** | 盈亏Tab → document.body.innerText |

**铁纪**：lotteryId=45 只用于北单，严禁混用竞足 lotteryId=90。逐场采集逐场分析，不可批量。innerText 一字不改。

### 北单赔率解读

- 北单「胜」= 让球方胜（上盘赢盘）
- 北单「平」= 让球方恰好赢N球（走水/让平）
- 北单「负」= 受让方不败（下盘赢盘）
- **北单三赔中最低的那个 = 市场最看好方向**（但不可盲信，需Edge验证）

### 五大核心理念

1. **赔率不是指南针，是对手方** — 先有独立判断，再用赔率找Edge
2. **双模并行** — 模式一（八维基本面）+ 模式二（实力盘定位），一致才出手
3. **Edge检测** — 模型概率 vs 市场概率，差价≥5%才出击
4. **渐进式分析** — 泊松λ计算 → 双变量泊松 → DC τ修正 → 让球映射
5. **进球数协同** — 让球方向与总进球数须逻辑一致

## 三、双模并行核心流程

```
┌─ 用户提出北单问题
│
├─ Step 1: 确认场次与数据采集
│  ├─ 获取北单列表页 → 提取matchID + 三赔
│  ├─ 逐场详情页6Tab提取（innerText one不改）
│  └─ 必发盈亏Tab（如有）
│
├─ Step 2: 模式一·八维预测法（基本面主导）
│  ├─ ①联赛势位 → ②主客战绩 → ③近期状态
│  ├─ ④交战历史 → ⑤相似对手 → ⑥初盘指数
│  ├─ ⑦盘中走势 → ⑧交叉综合判定
│  └─ 参考: references/09-prediction-methods.md
│
├─ Step 3: 模式二·实力盘定位法（盘口主导）
│  ├─ ①基本面梳理 → ②初盘合理性定位 → ③水位趋势
│  ├─ ④机构意图揣摩 → ⑤指向确定
│  └─ 参考: references/09-prediction-methods.md
│
├─ Step 4: 泊松模型基础计算
│  ├─ 市场校准λ（从北单三赔反推）→ references/03-market-edge.md
│  ├─ 统计λ（近3场×0.4+近6场×0.3+赛季×0.3）
│  ├─ 双变量泊松（ρ修正）→ references/01-poisson-models.md
│  ├─ DC τ修正（低比分精度提升）→ references/01-poisson-models.md
│  └─ λ_final融合 → 让球胜/平/负概率
│
├─ Step 5: 信号检查与Edge检测
│  ├─ 因子检查（4b/12/13等）→ references/04-common-mistakes.md
│  ├─ 联赛特异性阈值 → references/05-league-templates.md
│  ├─ 信号熔断检查 → references/10-review-methodology.md
│  └─ Edge检测（模型vs市场，≥5%出击）
│
├─ Step 6: 诱上阻上与盘口验证
│  ├─ 降水不升盘 → 诱上，走下盘
│  ├─ 升盘中高水 → 阻上，走上盘
│  ├─ CLV监控 → references/06-trap-clv.md
│  └─ 盘口技术手册 → references/07-handicap-guide.md
│
├─ Step 7: 进阶工具交叉验证
│  ├─ 必发交易量+冷热指数 → references/08-betfair-kelly-ou.md
│  ├─ 凯利指数排雷 → references/08-betfair-kelly-ou.md
│  └─ 大小球协同 → references/08-betfair-kelly-ou.md
│
├─ Step 8: 进球数预测
│  ├─ λ₁+λ₂ → 双变量泊松 → 总进球概率分布
│  ├─ 五大修正因子（G1赛程/G2伤停/G3天气/G4H2H/G5市场）
│  └─ 三步验证 → references/12-goal-prediction.md
│
├─ Step 9: 双模交叉验证
│  ├─ 两模一致 → ✅ 强确认，单选
│  ├─ 两模分歧+偏重判定命中 → 📌 并列[A]/[B] ➡ 偏重[A]
│  └─ 两模分歧+偏重不命中 → ⛔ 放弃
│
├─ Step 10: 复盘（赛后）
│  ├─ 比分命中率（精确/方向/偏差）
│  ├─ 信号准确率追踪
│  ├─ CLV追踪
│  └─ 联赛特异性规律 → references/11-review-insights.md
│
└─ 输出
```

### 偏重判定8级优先级

按顺序检查，命中即停：

| 优先级 | 条件 | 偏重 |
|:---:|:---|:---:|
| 1️⃣ | 因子4b触发（验证15/15） | → 偏模式二 |
| 2️⃣ | 实力盘偏离≥1档 | → 偏模式二 |
| 3️⃣ | 联赛=瑞典超/挪超/瑞典甲（100%可靠） | → 偏模式二 |
| 4️⃣ | H2H压倒性压制（≥4连胜或≥80%胜率） | → 偏模式一 |
| 5️⃣ | 保级/争冠战意差（赛季末5轮） | → 偏模式一 |
| 6️⃣ | 联赛=爱超/爱甲/J1/丹甲（信号不可靠） | → 偏模式一 |
| 7️⃣ | 模式二无信号 | → 偏模式一 |
| 8️⃣ | 以上均不命中 | → 放弃 |

## 六、操作铁规

### 执行层
- ❌ 禁止"北单三赔最低=市场看好方向"的机械判断
- ❌ 禁止输出精确比分（如2-1、1-0等）— 北单只出让球方向
- ✅ 允许输出总进球数预测（如2球/3球/4球）— 辅助维度
- ✅ 逐场采集逐场分析，不可批量
- ✅ 北单 lotteryId=45，严禁混用竞足 lotteryId=90
- ✅ 必须按北单编号顺序，不可跳场

### 判断层
- ✅ 八维交叉 → 维度表 → 方向
- ✅ 实力盘校准 → 偏离度 → 意图 → 方向
- ✅ 双模一致才出手，分歧则放弃
- ✅ 精选≤3场/日，忌广撒网
- ✅ 收盘后记录CLV，追踪判断质量

### 输出铁规
- ❌ 禁止"仅供参考"等免责套话
- ❌ 禁止批量输出多场——每场单独发送
- ✅ 终选前必须重新拉取欧指数据（因子4b会漂移）
- ✅ 进球数预测须与方向推荐做一致性检查
