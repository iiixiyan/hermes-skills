# 竞彩比分预测详细分析框架

> 本文件由 SKILL.md 在执行第 2-5 步时按需加载。

## 一、数据采集详细指标

### 1.1 联赛排名与积分形势
- 当前联赛排名、积分、胜平负场次
- 主客场分开统计：主场积分/客场积分、主场胜率/客场胜率
- 近期积分走势（近 6 轮积分曲线）

### 1.2 近 6 轮战绩
- 主队近 6 轮：总战绩 + 其中主场战绩
- 客队近 6 轮：总战绩 + 其中客场战绩
- 近 3 场额外标注（权重 40%）

### 1.3 进攻数据
- 场均射门数、射正率
- xG（期望进球）——近 6 场均值 + 赛季均值
- 反击进球占比、定位球进球占比
- 关键传球手助攻数据

### 1.4 防守数据
- 场均被射门数
- xGA（期望失球）——近 6 场均值 + 赛季均值
- 零封率
- 高压失误次数（导致射门的失误）

### 1.5 历史交锋
- 近 5 次对阵比分
- 让球盘口走势
- 战术相克特征（如某队面对高位逼抢时胜率骤降）

### 1.6 天气与场地
- 暴雨/高温/大风：影响技术流传导，利好防守方
- 人工草皮：影响球速和反弹，适应方占优
- 场地尺寸：小球场利好密集防守

## 二、伤停与战术影响

### 2.1 核心球员伤停评估
按影响等级分类：
- **致命级**：头号射手、组织核心、防线领袖、主力门将
- **重要级**：常规主力、战术支点
- **边缘级**：轮换球员、替补

评估替代者能力差距和战术调整可能性。

### 2.2 战术相克性分析
常见相克关系：
- 高位逼抢 → 克制 → 后场短传出球型
- 边路强攻 → 克制 → 三中卫翼卫回防慢
- 防守反击 → 克制 → 控球高压但防线身后空档大
- 密集防守 → 克制 → 缺乏远射/定位球能力的传控队

### 2.3 阵型变化预期
- 伤停导致的被迫变阵
- 针对性战术变阵的历史效果

## 三、战意与赛程体能

### 3.1 战意指数
按 1-5 级量化：
- 5 级：保级生死战、争冠天王山、德比战
- 4 级：欧战资格关键战、杯赛淘汰赛
- 3 级：争夺更好排名、杯赛小组赛
- 2 级：无欲无求但面对强队有表现欲
- 1 级：彻底无欲无求、已提前夺冠或降级

**触发异常权重的条件**：双方战意差 ≥ 2 级。

### 3.2 双赛体能
- 周中是否踢了欧战/杯赛
- 轮换幅度评估（首发更换几人）
- 客场旅行距离与休息天数
- 连续双赛的累积疲劳（第 3 场双赛体能下降更明显）

### 3.3 赛前动态
- 更衣室氛围（内讧传闻、将帅失和）
- 主帅赛前言论（是否有轮换暗示）
- 转会窗动态（核心球员被报价影响心态）

## 四、模型计算

### 4.1 基准预期进球
- 综合近 6 场 xG、赛季 xG、主客场系数，计算：
  - λ1（主队预期进球）
  - λ2（客队预期进球）
- 主客场系数参考：英超主场进球加成约 +15%~20%

### 4.2 状态权重
加权公式：

λ = (近3场均值 × 0.4) + (近6场均值 × 0.3) + (赛季均值 × 0.3)

### 4.2a 弱队客场λ修正（因子14·2026-05-26新增·P0强制）

**计算完基础λ后，按以下顺序应用修正：**

**步骤1：跨级杯赛修正（优先级最高）**
```python
if 跨级杯赛:
    if 低级别队是主队:
        λ_低 *= 0.75
        λ_高 *= 0.90
    else:
        λ_低 *= 0.65
        λ_高 *= 0.95
```

**步骤2：弱队客场λ系数（因子14）**
```python
if 客队排名比主队低 >= 5名:
    if 跨级杯赛:     # 跨级时已应用步骤1，此处不再重复
        λ_客 = λ_客  # 已修正
    elif 杯赛:
        λ_客 *= 0.70
    elif 联赛:
        λ_客 *= 0.80
```

**步骤3：解放者杯客场远征系数**
```python
if 联赛 == "解放者杯" and 跨国:
    if 飞行距离 > 2000km: λ_客 *= 0.65
    elif 飞行距离 > 1000km: λ_客 *= 0.75
    else: λ_客 *= 0.80
```

**步骤4：因子11弱队主场保底（应用步骤1-3后再检查）**
```python
if 主弱客强 and λ_主 < 1.20:
    λ_主 = max(λ_主, 1.0)
```

### 4.2b 跨级升降级附加赛λ修正（2026-05-29新增·P0强制）

**当识别为跨级升降级附加赛后（详见 `references/cross-league-playoff.md`），在步骤1-4之后应用以下修正：**

**步骤5：跨级附加赛识别**（在应用完步骤1-4后检查）
```python
if 主队联赛 != 客队联赛 and 比赛是升降级附加赛:
    # 背水一战系数
    if 主队是高级别保级队:
        λ_主 *= 1.30  # 背水一战+跨级实力差
    # 跨级防守压力
    if 客队是低级别升级队:
        λ_客 *= 0.75  # 低级别客场对高级别防守打折
    # 赛制修正（两回合制）
    if 首回合 == "0-0" and 主队是首回合客场:
        客队守平即晋级 → λ_客 *= 0.90
    # 因子4b降级
    if 因子4b触发:
        4b降级为P1参考（附加赛市场流动性低，信号可能失真）
```

### 4.3 攻防博弈结构判定
- **正路结构**：主队 xG > 客队 xGA 且 客队 xG < 主队 xGA
  → 实力差距清晰，大概率按预期走
- **失衡结构**：上述条件不满足
  → 存在不确定性，冷门概率提升

### 4.4 爆冷系数
满足以下任一条件触发 +1 级异常权重：
- 战意差 ≥ 2 级
- 核心射手/防线领袖伤停
- 保级队 vs 无欲无求强队
- 极端天气（暴雨/大风）
- 强队周中欧冠消耗 + 周末联赛间隔 < 72 小时

### 4.5 因子4b执行前强制检查（2026-05-28新增）

**在4b信号触发后、生成异常比分前，必须按以下顺序执行强制检查：**

**步骤1：失效条件检查**
```python
# 任一命中 → 4b降级为P1参考（不从P0执行）
if 比赛是杯赛决赛:
    4b_级别 = "P1"  # 决赛双方战意拉满+无主场优势，4b信号不可靠
    # 案例：3001 水晶宫vs巴列卡诺(欧协联决赛) 29升/0降但主胜打出
elif 比赛在中立场地:
    4b_级别 = "P1"  # 中立场地消除主场优势，4b是纯市场信号
    # 案例：同上（欧协联决赛为中立场地）
elif 主队近3场更换主教练:
    4b_级别 = "P1"  # 换帅效应（反弹/混乱）改变球队基本面
```

**步骤2：「价值回归」识别**（仅在4b触发主胜升赔方向时检查）
```python
# 场景：主胜升赔但实际是初赔过低的市场修正
if 主胜升 >= 20 and 主胜降 <= 2:  # 触发4b
    if 主队近10场胜率 >= 70% \
        and 主胜初赔 < 2.0 \
        and 平局降 >= 20 \
        and 客胜降 >= 20:
        判定 = "价值回归"  # 三方向都被压低=初赔过低的市场修正
        4b_级别 = "P1"    # 降级为参考，不从P0执行
        # 案例：3002 德尔瓦耶(近10场8胜,初赔1.83→2.02,29升/0降,平28降/客26降)
```

**步骤3：异常比分生成规则**
```python
if 4b_级别 == "P0":  # 通过检查
    异常比分必须包含反方向
elif 4b_级别 == "P1":  # 失效条件命中或价值回归
    异常比分可选包含反方向（不强制）
    正路信心只-1★（非P0的-2★）
```

## 五、异常因子挖掘

### 5.1 爆冷诱因
- 强队大幅轮换（联赛争冠无望 / 杯赛留力）
- 更衣室内讧或将帅失和
- 保级队背水一战 + 主场加成
- 历史交锋存在明显血脉克制
- 极端天气削弱技术流球队优势
- 关键球员赛前突发伤缺

### 5.2 极端赛果诱因
- 弱队防线核心缺阵 → 体系崩塌 → 大比分失利
- 强队需大量净胜球 → 放手狂攻 → 单方大球
- 双方均为互攻风格且无欲无求 → 对轰局
- 杯赛加时赛后的体能崩盘 → 尾声阶段连续失球
- 红牌/点球等意外事件改变比赛节奏

### 6.1 国际友谊赛特殊处理（2026-05-29追加）

### 6.1 国际友谊赛特殊处理（2026-05-29追加）

**特殊性**：
1. λ计算依赖的联赛数据不存在——球队没有"赛季均值"，只能用近几场国家队比赛历史数据
2. 排名差异 ≠ 实力差异——FIFA排名不反映即时状态
3. 让球穿盘率低——友谊赛中领先方常换人/收兵，1-0赢球输盘是典型模式
4. 综合实力分仍有效——阵容身价/平均年龄等数据可供参考

**处理建议**：
- 友谊赛信心不超过★★★☆☆（因联赛数据不可得）
- 因子12(一致降赔)在友谊赛中仍有效——一致降主胜30/0的强信号在主胜方向正确
- 让球预测偏向保守：主胜下让球倾向让平而非让胜（穿盘概率<50%）
- 异常比分中客胜方向的可信度低于联赛（友谊赛客队远征不会全力以赴）
- 因子2(战意)、因子9(争冠无望)等联赛专属因子不适用

**验证案例**：周四001 爱尔兰1-0卡塔尔(国际友谊赛)
- 因子12(一致降主胜30/0) → 主胜方向正确✅
- 让球(-1) → 1-0让平✅(穿盘率低预测正确)
- 无异常比分触发(客胜升30家=市场极度不看好客胜→异常客胜方向不可信)

### 6.2 国际友谊赛0-0/小球偏倚（2026-05-30新增）

**发现来源**：2026-05-29 5008 波黑0-0北马其顿

**核心规律**：国际友谊赛中，当双方实力接近（综合实力差≤10分）且让球方为-1浅盘时，0-0概率显著上升至约30-40%，远超联赛0-0概率(~10%)。

**原因**：
- 友谊赛双方都不会全力以赴
- 下半场大规模换人后进攻节奏被打断
- 双方主要目标为磨合阵容而非取胜

**修正式**：
```python
if 联赛 == "国际友谊" or 联赛 == "International Friendly":
    if 双方综合实力差 ≤ 10:  # 实力接近
        0-0概率 = 泊松P(0,0) × 2.5  # 0-0概率上调150%
        正常比分第1个应优先考虑0-0或1-1
    
    if 让球数 == -1 and 主胜初赔 > 1.70:
        让球优先: 让负 > 让平 > 让胜
        # 浅让友谊赛穿盘极难
```

**验证案例**：
| 场次 | λ1 | λ2 | 初赔 | 实际 | 预测改善 |
|:----|:--:|:--:|:----:|:---:|:--------|
| 5008 波黑0-0北马其顿 | 1.87 | 3.38 | 1.87 | 0-0 | 原预测1-0/2-0→应修正为0-0/1-1 |

### 6.3 国际友谊赛4b排除方向降级（2026-06-02新增）

**发现来源**：2026-06-01复盘，周一001保加利亚0-1黑山（百家平均客胜2.45→2.60，24家升/3家降触发4b排除客胜）。市场一致性排除客胜，但黑山客场1-0取胜。

**核心规律**：国际友谊赛中，4b排除方向信号可靠性显著低于联赛。原因：
- 友谊赛市场容量小，赔率调整的"信号噪声比"低
- 客胜升赔更多反映"战意不确定性"而非"市场看衰"
- 友谊赛资金流动性不足，少数大额资金即可扭曲信号

**处理规则**：
```python
if 联赛 == "国际友谊" or 联赛 == "International Friendly":
    # 4b排除方向信号降级
    if 因子4b_排除方向触发:  # ≥20家升赔且≤2家降赔
        4b_级别 = "P2参考"  # 仅信息标记
        异常比分可选包含反方向（不强制）
        正路信心仅-1★（非-2★）
    
    # 因子12一致降赔维持有效
    if 因子12_一致降赔触发:  # ≥20家降赔且≤2家升赔
        维持P0参考级别
        # 验证：1002(30降主胜✅) 1003(25降主胜✅) 1004(28降主胜✅)
```

**原因区分**：
- **4b（升赔排除）** = 赔率上升，反映的是"卖方增强"（更多人卖出该选项）→ 友谊赛中可能被单一因素（如客队阵容调整）驱动
- **因子12（降赔看好）** = 赔率下降，反映的是"买方涌入"（更多人买入该选项）→ 更真实的资金流动信号

**与挪超/瑞超降级的区别**：
| 联赛 | 降级幅度 | 原因 | 动作 |
|:----|:--------:|:----|:----|
| 挪超 | P0→P1+λ修正 | 高平率化解冷门 | 平局×1.25，异常比分含平局 |
| 瑞超 | P0→P1+λ修正 | 排名含金量低 | 同挪超 |
| 国际友谊赛 | P0→P2参考 | 市场噪声比高+战意不确定 | 仅信息标记，不强制修改 |

**验证案例**：

| 场次 | 4b信号 | 升/降 | 4b排除方向 | 实际 | 信号判定 |
|:----|:-------|:----:|:----------:|:---:|:--------:|
| 1001 保加利亚0-1黑山 | 排除客胜 | 24升/3降 | 客胜 | 客胜打出 | ❌ 假信号 |
| 7001 美国3-2塞内加尔(前日) | 排除客胜 | — | 客胜 | 主胜 | ✅ 有效(即排除方向正确) |

Note：友谊赛4b排除方向在有限的样本中呈现50%准确率，不足以作为P0决策依据。

### 6.9 窗口最终日V型恢复模式确认（2026-06-10新增）【注：应排列在 §6.8 之后】

**发现来源**：2026-06-09复盘，2场国际友谊赛（中国0-0泰国、匈牙利3-1哈萨克）。此前6/6已验证6/5主胜率回升至50%，6/8（周一）2场竞足友谊赛100%主胜，6/9（周二）2场50%主胜。

**核心规律**：FIFA友谊赛窗口主胜率呈清晰V型恢复模式：

| 窗口位置 | 主胜率 | 恢复阶段 |
|:--------|:-----:|:--------|
| 窗口中期（峰值低谷） | 0-20% | 低迷期 |
| 倒数第2天 | 33-50% | 恢复期 |
| 倒数第1天（最终日） | 50-100% | 正常化 |
| 窗口结束后 | 40-45%（正常）| 完全恢复 |

**处理规则确认**：
```python
if 联赛 == "国际友谊":
    if 处于窗口最终日（倒数0天）:
```

**核心规律**：预测生成前必须进行交叉验证，确认从59itou列表页/详情页获取的球队名与实际比赛匹配。未经核验的预测将产生无意义的输出。

**处理规则**：

```python
def verify_match_integrity(home_team_scraped, away_team_scraped, inner_text):
    """
    验证从59itou提取的队名数据是否与详情页内容一致。
    返回: (is_valid: bool, reason: str)
    """
    # 验证1：详情页innerText中应同时出现主客队名
    home_in_text = home_team_scraped and home_team_scraped in inner_text
    away_in_text = away_team_scraped and away_team_scraped in inner_text
    
    if not home_in_text and not away_in_text:
        return (False, f"两队名均未出现在详情页: {home_team_scraped}/{away_team_scraped}")
    if not home_in_text:
        return (False, f"主队名'{home_team_scraped}'未出现在详情页")
    if not away_in_text:
        return (False, f"客队名'{away_team_scraped}'未出现在详情页")
    
    # 验证2：检查是否有常见赛事关键词
    league_keywords = {"世界杯","友谊赛","世预赛","竞彩","瑞超","芬超","挪超",
                       "日职","韩K","英超","西甲","意甲","德甲","法甲","巴甲","阿甲",
                       "沙特联","比甲","荷甲","葡超","土超","俄超","奥甲","丹超",
                       "瑞典超","挪威超","瑞士超","希腊超","捷甲","克罗甲"}
    found_leagues = [kw for kw in league_keywords if kw in inner_text]
    
    # 验证3：检查prefix陷阱（详情页加载了完全不相关的比赛）
    if len(inner_text) > 500 and not found_leagues and not home_in_text:
        return (False, "数据可能来自不同比赛（prefix路由错误）")
    
    # 验证4：检查详情页是否包含对手联赛名（防止同matchID路由到不同比赛）
    # 世界杯比赛应包含"世界杯"或"World Cup"或双方国家队命名特征
    if "世界杯" in str(home_team_scraped + away_team_scraped):
        if "世界杯" not in inner_text and "国际赛" not in inner_text:
            # 可能是联赛比赛错误路由
            league_names = {"沙特联","英超","西甲","意甲","德甲","法甲","瑞超",
                           "芬超","挪超","日职","韩K","巴甲","阿甲","解放者杯"}
            found_wrong_league = [l for l in league_names if l in inner_text]
            if found_wrong_league:
                return (False, f"世界杯比赛详情页加载为联赛赛事({','.join(found_wrong_league)})")
    
    return (True, "验证通过")
```

**应用时机**：
1. 采集到详情页innerText之后、开始分析之前执行
2. 验证失败→停止当前场次预测，标记为"数据异常"
3. 切换前缀重试后验证仍失败→跳过该场，标注「❌数据异常，无法预测」

## 八、让球胜平负推导（每次预测必须输出）

### 8.1 让球数提取
从详情页欧指Tab数据中提取让球数。竞足列表页通常以「主队(-1)」格式显示让球数。

**提取方式**（从欧指Tab innerText）：
```python
# 查找 "-X)" 或 "(+X)" 模式
import re
match = re.search(r'\(([+-]?\d+)\)', ouzhi_text)
if match:
    让球数 = int(match.group(1))
else:
    让球数 = 0  # 默认平手盘
```

### 8.2 让球结果计算

**让球数含义（竞足官方格式）**：
- `(-1)` = 主队让1球（主队需净胜2+才算让胜）
- `(-2)` = 主队让2球（主队需净胜3+才算让胜）
- `(0)` = 平手盘（胜=主胜，负=客胜）
- `(+1)` = 客队让1球（主队受让，不败即让胜）

**计算公式**：
```python
def calc_rangqiu(主队进球, 客队进球, 让球数):
    """
    让球数 = 竞足官方让球数
    示例：-1=主队让1球, +1=客队让1球(主队受让)
    """
    净胜 = 主队进球 - 客队进球
    
    # 调整后净胜 = 主队净胜 - 让球数绝对值
    abs_hcp = abs(让球数)
    
    if 让球数 <= 0:  
        # 主队让球(<=0) 或 平手(=0)
        # 让胜：主队净胜 > 让球数绝对值
        # 让平：主队净胜 = 让球数绝对值
        # 让负：主队净胜 < 让球数绝对值
        if 净胜 > abs_hcp:
            return "让胜"
        elif 净胜 == abs_hcp:
            return "让平"
        else:
            return "让负"
    else:  
        # 客队让球(>0)，主队受让
        # 让胜：主队赢 或 平（主队不败）
        # 让平：主队输1球
        # 让负：主队输2球及以上
        if 净胜 >= 0:
            return "让胜"
        elif 净胜 == -1:
            return "让平"
        else:  # 净胜 <= -2
            return "让负"
```

**推导规则**：
- **正常比分**：双选，λ模型选最可能的2个比分覆盖
- **胜平负**：模型综合λ差+因子检查+市场信号独立判定，**非从比分反推**
  - 例：λ差小+因子12一致降赔→即使预测2-0，胜平负也可能=平（冷平预警）
  - 例：因子4b一致升主胜→即使预测2-1，胜平负也可能=负（冷门预警）
- **让球**：模型综合盘口水位+欧指信号独立判定，**非从比分+让球数换算**
  - 例：让球-1+比分2-0，但亚指水位不支持穿盘→让平而非让胜
- **异常比分**：双选，含异常路径标签
