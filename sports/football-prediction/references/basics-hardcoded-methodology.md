# 基本盘硬编码方法论 v1.0

> 用户铁律：所有基本盘(温度/天气/中立/评分/伤停/身价)必须硬编码在引擎规则链中。全部写死进去。

## v10.8g 架构：双层predict()

```
predict()  ← 包装函数: 应用λ修正到比分
  └─ _predict_raw()  ← 纯规则链: 返回硬编码比分+conf_mod
```

### predict() 包装函数核心逻辑

```python
def predict(...):
    # 1. 调用纯规则链
    h, a, rule, conf = _predict_raw(...)
    
    # 2. 重新计算λ系数(lm_h, lm_a)
    # 来源: form_signal(伤停/状态/实力差), temp(高温/低温), neutral(中立), is_r2plus
    
    # 3. 应用修正到整数比分
    if lm_h < 0.85:  h = max(0, h - 1)   # 重度削弱
    elif lm_h > 1.10: h = min(9, h + 1)  # 加强
    # lm_a同理
    
    return h, a, rule, conf
```

### 阈值说明

| λ系数范围 | 调整幅度 | 触发条件示例 |
|:---------|:--------|:------------|
| < 0.85 | -1球 | 伤停≥2级(0.85) + 高温≥30°C(0.90) + R2+(1.05) = 0.80 |
| 0.85-0.94 | 无调整 | 仅高温(0.90)单独 → 取整后不变 |
| 0.95-1.09 | 无调整 | 中立(0.95)、低温(1.05) → 取整后不变 |
| ≥ 1.10 | +1球 | 状态优势≥5(1.15) + 评分差≥0.5(1.10) |

### 实测覆盖

28场回测中 **0场**因基本盘修正跨越整数阈值。说明：
- 单一基本盘因子(高温/低温/中立)影响≤10%，取整后不改变比分
- 仅叠加多重因子(如伤停+高温+状态)才可能跨越阈值
- 如需激进修正可下调阈值至0.90/1.05

### 调用方式

```python
# 手动传入基本盘
h, a, rule, conf = predict(h="阿根廷", g="奥地利", ..., temp=31, neutral=1)

# 或通过form_signal传入结构化基本面
signal = {
    'injury_impact_h': 2,    # 主队核心伤缺
    'injury_impact_a': 0,
    'form_diff': 3,          # 状态差
    'strength_gap': 15,      # 实力差
    'lineup_known': True,
}
h, a, rule, conf = predict(h="阿根廷", g="奥地利", form_signal=signal, temp=31)
```

### 铁律

1. 每次预测前必须从新浪API footballMatchDetail 拉取environment字段解析温度
2. 每次预测前必须检查isNeutral字段(0=主场,1=中立)
3. 基本盘数据必须**写死在规则中**, 不允许作为可选参数或--browser依赖
4. 回测时必须传入基本盘参数, 不能只传赔率信号
