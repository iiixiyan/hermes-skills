## 三、v7.0.5新增规则

### R12-Trap-Draw（造热推平局）

**发现时间**: v7.0.3 → v7.0.4 迭代中，6035厄瓜多尔vs库拉索

**触发条件**: `hd + r2_high + c1≥30 + fd≥30 + fh>15 + o1<1.20`

**信号模式**:
- hd=True（客队状态好过主队）
- r2_high=True（平局端极端一致变动）
- c1≥30（30+公司下调主胜，诱导投注）
- fd≥30（FIFA排名差≥30位）
- fh>fa（主队FIFA排名高于客队 → 主队是弱队）
- o1<1.20（百家平均主胜极低 → 庄家压低赔率造热）

**输出**: 走平局(0-0/1-1)，方向偏平局

**案例验证**: 6035厄瓜多尔vs库拉索 — 41降主胜+45降平+40升客胜的极端一致信号，旧R12B-HD规则误判为4-1；R12-Trap-Draw正确输出1-1/0-0 ✅

**实现位置**: worldcup-predict-v10.py predict()函数，R12分支的第三个条件块，在P11-BigD-Far-A-Home/D之后

### 未采用规则：R2-MUSTWIN（战意差进球上修）

**条件**: 强队(FIFA≤15)积分落后(≤1分 vs ≥3分)且必须取胜 → λ×1.20~1.30

**状态**: ❌ 未集成

**根因**: 引擎predict()函数无积分差输入参数。新浪API footballMatchDetail虽有积分字段但需要二次验证。引入积分差需：
1. 在form_signal中添加 `points_h` / `points_a` 字段
2. 在worldcup-predict-all.py中从新浪API获取积分数据后注入
3. 添加R2-MUSTWIN规则分支
4. 荷兰vs瑞典(5-1)的3球偏差可由此修复

## 四、迭代方法论总结

### 正确回路（v7.0.0→v7.0.5验证）

```
① 基线回测 → 记录全部场次命中/偏差 → 不调参
② 识别偏差≥2球场次 → 逐场分析根因
③ 提出修复规则 → 写入规则链（单条规则，优先窄范围）
④ 全量回测验证 → 检查原有命中场次是否退步
⑤ 若退步 → 回退规则，缩小触发条件范围
⑥ 若无退步 → 推进到下一场偏差分析
⑦ 新增规则后 → 加入新比赛数据扩大覆盖
```

### 关键约束

| 约束 | 原因 |
|:----|:-----|
| 每次修改后必须**重新拉取实时数据**，禁用缓存 | 防止过拟合到缓存数据的时间点 |
| 新增规则触发条件**宁可窄不可宽** | 窄条件仅影响目标场次，宽条件可能破坏其他场次 |
| 方向正确优先于比分精确 | 方向正确(96.8%)是投注基础，比分精确(83.9%)是进阶 |
| 单条规则只修复1类偏差 | 复合规则难以调试和归因 |
| 规则优先级锚定 | 新规则放在对应P级别的最后，不影响高优先级规则 |

### 退出准则

迭代终止的判定条件（v7.0.5终止原因）：
1. □ 剩余偏差≥2球场次需要**外部数据源**才能修复 → 停止规则链修改
2. □ 偏差归因分析确认是**随机误差/不可控因素** → 标记盲区，不修改规则
3. □ 修复后回归检测导致原准确场次退步 → 回退规则

v7.0.5终止原因: 荷兰vs瑞典(3球偏差)需要积分差数据（外部输入），引擎当前无此参数。引入积分差是feature升级、非规则调优 → 等待下一版本。
