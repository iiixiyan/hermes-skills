# 联赛进球率校正方法论

## 背景

`league-predict.py` 的 `compute_team_stats()` 函数通过拉取 sporttery API 历史数据获取每支球队的赛季平均进球/失球值。但存在以下局限：

1. **TEAM_MAP覆盖**: 未映射的联赛(之前缺韩K联/瑞典超)使用默认值 avg_gf=1.25
2. **数据量有限**: sporttery API 只返回约30天内的比赛, 赛季初数据少时λ偏保守
3. **近期状态**: 近3场进球/失球趋势被赛季均值平滑掉, 需要用近3场 vs 赛季均值做偏差检测

## 近期状态修正 (v10.28 §3.1)

在 `apply_all_corrections()` 中新增 `3.1 近期进球状态修正` 段:

```python
# 攻击爆发: 近3场进球 > 赛季×1.3
if h_gf > 0 and h_recent_gf3 > h_gf * 1.3:
    boost = 1.12 if h_recent_gf3 > h_gf * 1.5 else 1.08  # 极度暴发 vs 普通暴发
    lambda_h *= boost

# 防守崩溃: 近3场失球 > 赛季×1.3
if h_ga > 0 and h_recent_ga3 > h_ga * 1.3:
    lambda_a *= 1.10  # 对手进球+10%
```
