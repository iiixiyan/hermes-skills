# v10.7h Bug修复记录 (2026-06-20)

## Bug 1: `neutral == 1 and h == ""` 条件永不触发

**现象**：所有世界杯比赛（均为中立场地）的 λ主×0.95 修正从未生效。

**根因**：
```python
# 旧代码 (v10.7g)
if neutral == 1 and h == "":
    lambda_h_mod *= 0.95
```
`h`是主队名称字符串（如"荷兰"），永远不等于`""`。此条件每次都是 False。

**修复**：
```python
# 新代码 (v10.7h)
if neutral == 1:
    lambda_h_mod *= 0.95
```

## Bug 2: `predict_with_basics()` 不转发基本盘参数

**现象**：通过 `predict_with_basics()` 调用时，neutral/weather/temp 使用默认值0，基本盘修正从未执行。

**根因**：
```python
# 旧代码 (v10.7g) — 调用 predict() 时漏传 neutral/weather/temp
h_base, a_base, rule, conf = predict(h, g, fh, fa, o1, o3, r1, c1, r2, c2, r3, c3, rd)
```

**修复**：
```python
# 新代码 (v10.7h)
h_base, a_base, rule, conf = predict(h, g, fh, fa, o1, o3, r1, c1, r2, c2, r3, c3, rd, neutral=1)
```

## 启示

1. 基本盘规则写死到引擎但不验证是否真的执行了，等于没写。
2. `if x and y == ""` 模式在Python中极易产生永不触发的条件。
3. wrapper函数（predict_with_basics）必须显式透传所有参数，不能依赖默认值。
