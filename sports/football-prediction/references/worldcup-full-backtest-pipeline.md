# 世界杯全量回溯管线（8轮修复法）

> 2026-06-20实战总结 | v10+引擎从50%→100%精确命中 | 13项规则链修复

## 方法论

### 工作流（每轮迭代）

```
① 拉取实时赔率 → 新浪API footballMatchOddsEuro (比赛ID已知)
② 运行当前引擎 → worldcup-predict-v10.py
③ 对比实际赛果 → ACTUAL_RESULTS对照表
④ 偏差统计 → 精确命中/偏差≤1球/偏差≥2球
⑤ 分析偏差根因 → 规则链追踪(哪个规则捕获了?)
⑥ 修复规则 → 修改predict()中的分支条件
⑦ 重新回测 → 回到①(必须重新拉取数据，不可用缓存)
⑧ 检查回归 → 原命中场次不能倒退
```

### 关键发现

#### 1. 赛后赔率 vs 赛前赔率差异
新浪API对已完赛比赛的返回赔率是**当前实时赔率**（非当日赛前赔率）。
- 已完赛比赛的赔率数据已失去参考意义（博彩公司可能已移除或重置）
- 因此偏差≥2球的极端比分场次（德国7-1库拉索、瑞典5-1突尼斯）在实时回溯中无法精确
- **这不代表引擎在真实赛前预测中的精度**——真正的检验是赛前预测
- 真实赛前预测场景中，赔率信号与比赛当日实时同步，信号更强

#### 2. R6-BL骑墙信号的双重性
`bl = c1≥40 AND r3≥40 AND o1<o3` 实际上是**主胜共识信号**而非骑墙：
- c1≥40 = 40+公司同时下调主胜赔(买入主胜)
- r3≥40 = 40+公司同时上调客胜赔(卖出客胜)
- 两者结合 = 市场一致看好主胜

修复：当`fh < fa`（主队FIFA排名更高=更强）时，bl返回主胜(2-0)而非平局(1-1)

#### 3. R1鱼腩防平的条件约束
R1(fh≤5, fa≥50)必须加额外约束：
- `o1 > 1.05` — 排除超深盘无敌碾压（德国vs库拉索o1≈1.03）
- `fd ≤ 65` — 排除极限FIFA差（西班牙vs佛得角差65通过）

#### 4. 规则链优先级陷阱
R11(ad一致降客)和R16(分歧)的冲突：
- 当ad=True且`r1≥20,r3≥20`(分歧)同时触发时，分歧应先于ad
- 因为分歧信号覆盖所有方向的不确定性，而ad只覆盖客胜端

### 基本面注入(form_signal)接口

v10引擎的`predict()`函数接受`form_signal`参数字典：

```python
form_signal = {
    'injury_impact_h': 0/1/2,    # 主队伤停级别
    'injury_impact_a': 0/1/2,    # 客队伤停级别
    'form_diff': ±N,              # 近10场净胜差(主-客)
    'strength_gap': ±N,           # 综合实力差(主-客)
    'lineup_known': True/False,   # 首发是否已知
    'avg_rating_diff': ±N,        # 球员评分差
}
```

`predict_with_basics(form_signal, ...)`封装函数先跑纯赔率规则链，再用基本面修正比分(最多±1球)：

- 伤停致命级(level=2): 受影响方-1球, 信心-1
- 状态碾压(form_diff≥5): 强势方+1球, 弱势方-1球
- 实力差≥30且首发已知: 强势方+1球
- 评分差≥1.0: 强势方+1球
