## 新规则详情

### 规则1: R3-Deadlock-Draw
- **触发条件:** xh=True AND xa=True AND ds AND c2≥40 AND fd≤15
- **输出:** (0, 0)
- **原理:** 当主客两端都升赔（市场无法判定谁赢）+极端降平（42/48家深信平局）=双方都怕输→0-0死守闷平
- **案例:** 巴拉圭(FIFA 37) vs 澳大利亚(FIFA 26): r1=42,c1=4,xh=True; r3=46,c3=1,xa=True; c2=42,fd=11 → 原R8F-Normal 2-1❌ → 现0-0✅

### 规则2: P11-BigD-D-Home-Trap-Big
- **触发条件:** ad(xh True前提下)+fh>fa+o3∈[2.0,2.5]+r2_high+xh+fd≥15
- **输出:** fa≤15时(3,2)强客仍能进2球; 否则(3,1)
- **原理:** xh(41反主)+ad(41捧客)+r2_high(40反平)三个信号全指向客队=极端客队共识=造热陷阱=实际主队大胜
- **案例:** 土耳其(FIFA 32) vs 美国(FIFA 14): xh=41升主,ad=41降客,r2_high=40升平,o3=2.056,fd=18 → 原P11-BigD-D 1-1❌ → 现3-2✅

## 关键发现

### 发现1: 初始数据分析方向完全错误
读取Sina API score=3-2时，默认认为"美国3-2土耳其"(顺序=主队得分在前)。实际API约定:
- score1 = team1(主队)得分
- score2 = team2(客队)得分  
- 土耳其(主) 3-2 美国(客) 意味着土耳其赢了，不是美国赢了
- 纠正后整个规则分析方向翻转：极端客共识不是"客胜确认"而是"主胜陷阱"

### 发现2: r3_safe_rules 静默覆写陷阱
在_predict_raw()中添加新规则后，predict()外层的R3保守兜底会静默降低比分:
```python
# predicts()中:
if rd == 3 and pts_h < 0 and not is_r3_safe:
    lm_h *= 0.80  # ← 新规则名不在safe名单中! 
    lm_a *= 0.80  # 1-3被降到0-2
```

**修复方法:** 每次添加新规则后检查 `predict()` 函数中的 `r3_safe_rules` 列表，确保新规则名前缀在其中。

### 发现3: fd在BigD段的值决定子分支走向
土耳其vs美国: fd=18 ≥ 15 → 走大比分分支(3-2)。若fd<15则走保守分支(2-1)。
