### BUG-4: R6-BL-Home-Trap 触发过于宽泛 — 中游主队真实方向误判为造热

**场次:** 刚果金 3-1 乌兹别克 (R3)  
**引擎:** R6-BL-Home-Trap → (0,0)  
**实际:** 3-1 偏差3❌

**信号数据:**
- fh=46, fa=57, fd=11 (主队FIFA更好,但差距很小)
- r1=0, c1=47, r2=44, c2=2, r3=47, c3=0
- 信号: hd=True(47降主), xa=True(47升客), bl=True(47降主+47升客+o1<o3), r2h=True(44升平)

**规则链追踪:**
1. R3专用规则 → pts_h=-1, 跳过
2. P11 → fh=46>10, 跳过
3. P11-B → fh=46>15, 跳过
4. BigD → r2=44>=40✓, c3=0<40✗, 跳过
5. R2+各项 → 均不满足
6. **R6骑墙(L.884-938):** bl=True, fh=46<57=fa, fd=11<20, 进入fh<fa分支(L.907)
7. L.910: hd=True但o1=1.681>1.50, 跳过R6-BL-HD-Boom
8. **L.923:** hd=True✓, r2_high=True✓, o1=1.681>1.50✓ → **R6-BL-Home-Trap(0,0)**

**根因分析:**
`hd and r2_high and o1 > 1.50` 条件太宽。此规则本为**精英主队造热**场景设计（比利时vs伊朗: fh=10,fa=22,实际0-0）。但对**中游主队**（fh>15）且fh<fa（主队FIFA排名更好）时，hd信号(47家降主胜)是真实市场方向，非造热。

**关键区别:**

| 场次 | fh | fa | fd | hd | bl | r2h | o1 | 实际 | 应有 |
|:----|:-:|:-:|:--:|:--:|:--:|:---:|:--:|:----:|:----:|
| 比利时vs伊朗 | 10 | 22 | 12 | ✓ | ✓ | ✓ | 2.20 | 0-0 | 造热✅ |
| **刚果金vs乌兹别克** | 46 | 57 | 11 | ✓ | ✓ | ✓ | 1.68 | **3-1** | **真实❌** |

**修复方案:**
在L.923之前插入中游主队真实方向分支:
```python
# R6-BL-Trust-HD: 中游主队(fh>15)+主队FIFA更好(fh<fa)+接近(fd≤15)+全部方向信号一致→真实方向
# 刚果金vs乌兹别克(R3): fh=46,fa=57,fd=11,hd+bl+r2h+c1≥40,实际3-1
if fh < fa and hd and r2_high and bl and c1 >= 40 and fd <= 15 and o1 > 1.50 and fh > 15:
    if o1 < 2.0:
        # 主队买入信号强烈,主胜2球左右
        return (3, 1, f"{rule_prefix}R6-BL-Trust-HD", 3 + conf_mod)
    return (2, 1, f"{rule_prefix}R6-BL-Trust-HD-Small", 2 + conf_mod)
```
预期: 刚果金3-1✅(偏差3→0)

### BUG-5: R2-GRP-Rot 未加入r3_safe_rules → R3保守兜底误降比分

**场次:** 克罗地亚 2-1 加纳 (R3)  
**引擎:** R2-GRP-Rot → (1,0)  
**实际:** 2-1 偏差2❌

**根因分析:**
_predict_raw()返回(2,1)后,在predict()中:
- pts_h=-1 (无积分数据)
- rule_name='R2-GRP-Rot' 不在r3_safe_rules名单中
- is_r3_safe = False
- lm_h *= 0.80 → adj_h从2降到1
- lm_a *= 0.80 → adj_a从1降到0
- 最终输出(1,0), 偏差2球

R2-GRP-Rot本身已是R2+出线战意规则,其(2,1)比分已经考虑了克罗地亚FIFA#13 vs 加纳FIFA#65的实力差。再被R3保守修正×0.80就过度了。

**修复方案:** 将'R2-GRP-Rot'和'R2-GRP-ARot'加入r3_safe_rules:
```python
r3_safe_rules = ['R3-', 'P11-Hot-H-Qual', 'R4-', ..., 
                 'R2-GRP-Rot', 'R2-GRP-ARot',  # 新增
                 'R8F-Normal-Close']
```
预期: 克罗地亚2-1→(2,1)→偏差2→0✅

## 经验总结

1. **R3无pts数据=引擎盲区:** 刚果金(偏差3)和克罗地亚(偏差2)均因pts未知走保守分支。每周cron预测前应尽可能传入pts数据
2. **R6-BL-Home-Trap需分层:** 精英主队(fh≤15)可造热,中游主队(fh>15)且fh<fa且信号一致时不可造热
3. **R3安全名单需补充:** R2-GRP-Rot、R2-GRP-ARot应加入名单避免保守修正误降比分
4. **λ模型在信号冲突时失误率高:** 芬超2场偏差≥3的比赛均为λ方向与欧指信号冲突,需额外检查机制
5. **方向准确率较高(75%世界杯/60%整体):** 表明大方向判断流程基本可靠,主要问题是比分precision
