### 第2步：信号诊断（核心 — 不要猜信号值）

#### 2a. 批量信号矩阵（对所有misses）

对于偏差≥1球的比赛，写诊断脚本一次性打印全部信号变量+规则链入口条件：

```bash
# 参考模式：debug_misses.py
# 对每个matchId从新浪API实时拉取欧赔数据
# 计算引擎中所有使用的信号变量
```

```python
# 关键模式 — 对每个偏差场:
odds = fetch_odds(mid)  # 新浪API实时拉取
# 计算全部信号变量——不遗漏任何一个引擎中使用的：
xh   = r1>=40 and c1<=5        # 极端升主胜
xa   = r3>=40 and c3<=5        # 极端升客胜
xa_light = r3>=35 and c3<=3    # 温和升客胜
hd   = c1>=25 and r1<=10       # 一致降主胜
ad   = c3>=25 and r3<=10       # 一致降客胜
df   = c2>=15                  # 平降(弱)
ds   = c2>=25                  # 平降(强)
bl   = c1>=40 and r3>=40 and o1<o3  # 骑墙
he   = xh and c3>=20           # 升主+降客组合
ra   = he and o3<o1            # 极端升主+降客+客赔更低
ts   = xh and df and c3>=40 and fh<fa  # 顶级强队信号
it   = he and o1<o3 and not (fh<fa)   # 造热信号
hb   = abs(r1-c1)<=5 and r3>=35 and r1>=20  # 双向极端
r2_high = r2>=30 and c2<=10    # 排除平局

# 输出一行诊断:
print(f"{h}vs{g}: pred{xh}实际{ah}-{aa} | "
      f"o1={o1} o3={o3} | "
      f"r1={r1} c1={c1} r2={r2} c2={c2} r3={r3} c3={c3} | "
      f"fd={fd} fh={fh} fa={fa} | "
      f"xh={xh} xa={xa} hd={hd} ad={ad} df={df} ds={ds} | "
      f"bl={bl} ra={ra} r2h={r2_high} hb={hb}")
```

**分析原则：**
- 对比偏差场和命中场的信号差异
- 查看偏差场进入了哪条规则分支
- 检查规则条件中哪个判断偏离了实际方向
- 从诊断表格中找「类似信号命中场」参考正确输出

#### 2b. 规则链手动追踪（核心手法）

对于每个偏差场，拿到信号矩阵后，**逐行追踪引擎的if/elif链**，找出进入的是哪个分支：

```python
# 读predict()函数，逐行对比信号值
# 比如R2块检查:
if fd >= 45 and o1 < 1.20:
    # 检查跳过条件: r1>=25 and c3>=20? 不满足→不走
    # 检查新跳过条件: is_r1 and not xh and xa and fh>fa? 
    # 不满足→继续走
    # elif o1 < 1.05: ... 不满足
    # elif o1 < 1.10: ... 不满足
    # else: R2-DeepS → 就是这里错了！
```

**常见错误模式：**
- 信号变量计算错误（以为xh=True，实际上r1<40）
- 跳过了正确的规则（跳过条件没覆盖这个case）
- 方向判断缺失（规则没检查`fh>fa`，默认走主队方向）
- 子分支顺序错误（更具体的规则在后，被前规则拦截了）

### 第3步：定位根因

| 诊断角度 | 问 | 示例 |
|:--------|:---|:-----|
| 信号冲突 | xh+xa同时触发？ | 比利时vs埃及: xa+hd触发但首轮应平局 |
| 排名反转 | fh>fa但规则按主强处理？ | 乌兹别克1-3哥伦比亚 |
| 阈值边界 | fd接近阈值但方向错误？ | 巴西vs摩洛哥fd=1→不应走强队方向 |
| 信号缺失 | 首轮+客强的平局分支缺失 | P11-Hot-B-D1新增 |
| 分支顺序 | 规则A拦截了规则B的场次？ | P11-Big-Draw早于R8 |
| **方向反转** | **主弱(fh>fa)时规则按主强方向走？** | **伊拉克vs挪威: R8F-Big 3-1→需1-3** |
| **弱队被高估** | **弱队被深盘但仍走碾压？** | **沙特vs乌拉圭: fd=45但主FIFA58** |
| **信号不可靠域** | **超级弱旅(fh≥70)时市场信号反指？** | **加纳vs巴拿马: r2≥40+c3≥40指向客胜但主胜** |

### 第4步：修复规则

在 `worldcup-predict-v10.py` 中：
1. 在对应规则分支中添加新条件分支
2. **优先用已有信号变量**（xh/xa/hd/ad/df/ds/bl/ra）
3. 新分支注释写明：触发条件 + 案例场次 + 实际赛果
4. 保留旧分支作为fallback

**修复铁律：**
- ❌ 不修改已有规则条件（会破坏命中场次）
- ✅ 在已有分支中添加更具体的子分支（优先匹配）
- ✅ 新分支放在旧分支之前（Python从上到下匹配）
- ⚠️ 新分支条件要足够窄，避免误抓其他场次

### 第5步：回归验证

每次修复后立即运行：
```bash
python3 worldcup-full-backtest.py
```

检查：
- ✅ 偏差场修复 → 精确命中
- ✅ 原命中场无回归（regression = 0）
- ⚠️ 若回归 → 加窄条件（如 `and not hd`）

| 回归症状 | 根因 | 修复 |
|:--------|:-----|:-----|
| 新分支太宽抓了不该抓的场 | 条件不够具体 | 加 `and not hd` / `and fd>=40` / `and is_r1` |
| 新分支覆盖了后面的分支 | 顺序错误 | 确保新分支在旧分支之前 |
| 信号本身有歧义 | 定义不精确 | 用复合条件而非单信号 |
