### §4.1 共同对手桥接法（Transitive Score Inference · v3.8 新增）

> **数学基础**：Bradley-Terry 配对比较 + Glicko-2 传递排名 — 以共同对手为标尺
> 实战案例见 `references/bridge-method-examples.md`
> 文彩54接入见 `references/external-reference-wencai54.md`

#### 核心思想

```
已知 A vs C = 2-1, B vs C = 2-4
→ 以C为标尺：A和B攻击力相同(2球)，但A防守远强于B(丢1 vs 丢4)
→ 推断 A vs B：A应占优，A进球 > B进球
```

**传统λ完全依赖自身统计数据 → 忽略了"通过第三方比较"的信息。**

#### 计算步骤

**Step 1：找共同对手**

从战绩Tab的「近10场」完整列表中提取：
- 主队近10场**全部**对手（不分主客场）
- 客队近10场**全部**对手（不分主客场）
- 取交集 → 共同对手集合 {C₁, C₂, ...}
- ⚠️ **排除杯赛**（英锦赛/足总杯等，轮换阵容不可比）

若共同对手 = 0 → λ_transitive不可用，γ=0
若共同对手 ≥ 1 → **必须使用桥接法**（即使仅1桥）

**主客场校准**：桥接前统一校准到中立场地
```
主场→中立：进球×0.85, 失球×1.15
客场→中立：进球×1.15, 失球×0.85
```

**Step 2：逐桥计算传递得分**

对每个共同对手 C：
```
A主场 vs C: (GA, GC_A)   ← 主队进球GA，C的进球GC_A
B客场 vs C: (GB, GC_B)   ← 客队进球GB，C的进球GC_B

→ C的防守强度评分：
   def_C_vs_A = GC_A / GA   （C对A每球丢多少，越小=防越强）
   def_C_vs_B = GC_B / GB   （C对B同理）

→ 桥接推断（以C为中介，推A对B）：
   A_goals_via_C = GA × (GC_B / GB)      ← A的进攻力 × C对B的"漏洞系数"
   B_goals_via_C = GB × (GC_A / GA)      ← B的进攻力 × C对A的"漏洞系数"
```

**直观理解**：如果C对B丢了4个而对A只丢1个 → C在B面前防守更差 → A面对"更差的C"也能进更多球。

**Step 3：多桥降噪**

```
A_transitive = median(所有桥的 A_goals_via_C)
B_transitive = median(所有桥的 B_goals_via_C)
```

**上限截断规则（v3.9新增）**：
```
若单桥值 > 3.0 → 截断为3.0，标记"极值截断"
若 >50%的桥被截断 → γ权重降一档（如γ=0.25→0.15）
若全部桥被截断 → 仅保留最小非截断桥，γ=0.08
若仅1桥且被截断 → γ=0（桥接法不可用，数据异常）
```

**为什么用中位数**：单桥受限于C的状态波动（主力伤停、红牌等），取中位数消除极端值。加上限截断防止数值爆炸（如双方都被同一强队碾压时的桥值失真）。

**Step 4：转为λ**

```
λ_transitive_A = A_transitive × 主场修正系数
λ_transitive_B = B_transitive × 客场修正系数
```

#### 多λ融合

```
λ_final_A = α × λ_market + β × λ_stats + γ × λ_transitive

默认权重：α=0.5, β=0.35, γ=0.15
- 共同对手≥4个 → γ=0.25 (多桥极可靠)
- 共同对手=3个 → γ=0.20
- 共同对手=2个 → γ=0.15
- 共同对手=1个 → γ=0.08 (单桥降权，但仍使用)
- 共同对手=0个 → γ=0 (不可用)
- λ_transitive与λ_stats方向一致 → γ+0.03
- λ_transitive与λ_stats方向矛盾 → γ-0.03 (最低0.05)
```

#### 实战公式模板（简化版）

```
单桥公式：A_goals = GA × (GC_B / GB)
         B_goals = GB × (GC_A / GA)
多桥降噪：取中位数，>3.0截断
零封兜底：GC=0时，用0.5替代
杯赛排除：英锦赛/足总杯等不参与桥接
```

详细计算示例见 `references/bridge-method-examples.md`。

#### ⚠️ 注意事项

- **零封处理**：当 C 被零封（GC=0），用 0.5 替代避免除零
- **极端值**：单桥得分 > 3×其他桥中位数 → 视为异常，降权0.5倍
- **主客场校准**：所有桥接推断都基于 A主场 vs B客场 的场景，如需中性场地需去主场修正
- **时间窗口**：只用近10场内的共同对手，超出10场的信息噪音太大
- **杯赛 vs 联赛**：杯赛和联赛的数据不混用（杯赛轮换多）
