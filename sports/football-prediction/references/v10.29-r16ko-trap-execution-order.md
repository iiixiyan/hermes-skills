# R16-KO-TRAP vs R32-KO-Defensive 执行顺序

## 规则栈

```
R16-KO-TRAP (⑤)          ← 最高优先级: 检测hd+xa+r2h三极端
    ↓ 不触发
R32-KO-Defensive (③)     ← 次优先级: 检测hd+r2h(无xa)
    ↓ 不触发  
R32-KO-SUPER-GAP (④)     ← 第三优先级: fd≥40
    ↓ 不触发
R6-BL-XA-Exempt (③bis)   ← fd≤10 + xa + hd
    ↓ 不触发
R10H-Other / R6-BL-Trap   ← 常规淘汰赛规则链
```

## 关键规则

| 规则 | 触发条件 | 输出 | 位置 |
|:----|:---------|:----|:-----|
| **R16-KO-TRAP** | rd=0 + hd(c1≥40+r1≤5) + xa(r3≥40) + r2h(r2≥30+c2≤10) + 10≤fd≤40 + fa>20 | **(1,2)** conf=4 | **最前** |
| **R32-KO-Defensive** | rd=0 + hd + r2h + fd<40 + fa>20 | (2,0) conf=3 | 第二 |
| **R32-KO-SUPER-GAP** | rd=0 + fd≥40 + hd + r2h + fa>20 | (1,1) conf=2 | 第三 |

## 关键区分

- R16-KO-TRAP **检查 xa 是否触发** — 当 hd+xa+r2h 三者全开 = 诱盘信号(1,2)
- R32-KO-Defensive **不检查 xa** — 仅 hd+r2h = 防冷信号(2,0)
- 因此 R16-KO-TRAP 必须在 R32-KO-Defensive **之前**执行，否则 xa 场景会被防冷规则错判为主胜方向
