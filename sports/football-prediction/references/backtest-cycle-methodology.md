# Backtest Methodology (2026-06-25更新 — 含v10.8k全量循环回测流程)

## ⚠️ 核心原则：固定测试集 → 过拟合，不可用于反哺skill

**2026-06-20方法论更正**：废除固定测试集(fixed_testset.py)作为验证手段。
固定集：用同一批16场比赛反复调参直到100% → **过拟合**，不能代表引擎的真实预测能力。

## 正确的反馈回路（Forward-Testing Only）

```
赛前预测 → 等待赛果 → 赛后对比 → 有偏差才修复 → 修复后用于下一轮新比赛
                             ↓
                      下一轮新比赛(而非旧比赛)
```

**每步含义：**
1. **赛前预测** — 用当前引擎对未开赛比赛输出比分
2. **等待赛果** — 比赛完赛后采集实际比分（数据源：okooo开奖页）
3. **赛后对比** — 比对预测vs实际，分析偏差模式
4. **有偏差才修复** — 只有偏差≥2球时才修改规则链
5. **修复后用于下一轮新比赛** — 绝不用修复后的引擎去回测旧比赛（那是对已知赛果调参）

### 步骤3：循环修复

```
1. 运行全量回测 → 列出所有偏差≥2球场次
2. 对每个偏差场次：
   a. 读取预测文件中的8参数数据(o1,o3,r1,c1,r2,c2,r3,c3,FIFA排名)
   b. 在_predict_raw()中手动追踪信号变量(xh/xa/hd/ad/bl等)
   c. 定位触发的不正确规则分支
   d. 添加新规则分支或修改现有条件
3. 回归检查：重新运行全量回测，确认：
   - 新修复的场次偏差降为0-1球 ✅
   - 原来正确的场次不退化(回归测试)
4. 重复直到偏差≥2球=0
```

### 追踪技巧

- 使用 `grep -n "规则名\|function\|return ("` 定位代码位置
- 信号变量定义在_predict_raw()开头：
  ```python
  xh = r1 >= 40 and c1 <= 5      # 极端升主胜
  xa = r3 >= 40 and c3 <= 5      # 极端升客胜
  hd = c1 >= 25 and r1 <= 10     # 一致降主胜
  ad = c3 >= 25 and r3 <= 10     # 一致降客胜
  bl = c1 >= 40 and r3 >= 40 and o1 < o3  # 骑墙
  r2_high = r2 >= 30 and c2 <= 10 # 升平极端
  df = c2 >= 15                   # 平降
  ds = c2 >= 25                   # 强平降
  ```
- 用 `python3 -c "import ...; v10.predict(...)"` 快速验证单场
- pts积分从R1结果计算: `R1胜=3分,平=1分,负=0分`

## 队伍实力对比系统（团队实力分：0-100）

**2026-06-20新增**：综合5维数据的队伍实力评分

| 维度 | 权重 | 说明 |
|:----|:----:|:-----|
| FIFA排名 | 30% | 排名越低→分越高 |
| 59itou综合实力分 | 25% | 0-100分的球队综合评估 |
| 球员总身价 | **20%** | 基于Transfermarkt公开数据，对数缩放 |
| 近期状态(10场) | 15% | 近10场胜率×10 |
| 球员评分 | 10% | 基于titan007平均评分(7.0=50分) |
**伤停惩罚** | -5分/人 | 每个核心伤停-5分 |

**身价对照表**：`scripts/team_strength.py` 的 `SQUAD_VALUES` 字典（预置32强+其他队伍数据）

**进球修正规则：**
| 实力差 | 等级 | 进球修正 |
|:-----:|:----|:--------|
| ≥25 | 碾压 | **+1球**(v10.8从+2下调, 碾压+2过激) |
| ≥8 | 明显优势 | **+1球** |
| ≥3 | 小幅优势 | 0 |
| -3~3 | 接近 | 0 |
| ≤-8 | 明显劣势 | -1球 |
| ≤-25 | 被碾压 | -2球 |

**集成方式：** 在引擎规则链跑完后调用 `apply_strength_to_prediction()`:
```python
from team_strength import compute_team_strength, apply_strength_to_prediction

# 先跑赔率规则链
h, a, rule, conf = predict(...)

# 再用实力对比修正
strength = compute_team_strength(fh=8, fa=34, gap_59itou=26,
                                 h_name='荷兰', a_name='瑞典')
h2, a2, reason = apply_strength_to_prediction(h, a, strength)
```

**⚠️ 冷门保护（v10.8新增）：**
当纯赔率模型检测到冷门/平局信号时(`P11-/R13/R10A-/R10D-/R10-Trap/R16-`)，队伍实力对比**仅增强信心不改比分**。
这是为了防止实力对比系统覆盖正确的冷门预测（如葡萄牙1-1民主刚果、卡塔尔1-1瑞士等案例已验证有效）。

## 基本盘参数实际影响力（⚠️非常重要的发现——2026-06-20）

当前引擎的天气/温度/中立修正用的是λ系数(×0.90~1.05)，
**在实际取整后对比分零影响**：

| 场景 | λ修正 | 原始λ | 修正λ | 取整比分 | 实际改变? |
|:----|:-----:|:-----:|:-----:|:--------:|:---------:|
| 中立(×0.95) | 2.0→1.9 | 2 | 1 | 2-0 | ❌ 不变 |
| 高温(×0.90) | 2.0→1.8 | 2 | 1 | 2-0 | ❌ 不变 |
| 低温(×1.05) | 2.0→2.1 | 2 | 2 | 2-0 | ❌ 不变 |

**修正方案**：应将λ系数修正改为**比分级直接修正**（而不是百分比λ）：
- 中立场地 → 主队-1球
- 高温≥30°C → 双方各-1球
- 低温≤15°C → 客队-1球
- 评分差≥0.5 → 强队+1球（当前阈值1.0太高）
