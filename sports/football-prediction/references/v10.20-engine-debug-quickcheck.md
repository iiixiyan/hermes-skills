### 3. 规则链路由检查

同一偏差模式可能来自不同规则链：

```python
# 不要只看比赛名称找规律
# ✅ 正确: 按 rule_name 分组统计
deviation_groups = defaultdict(list)
for match in deviations:
    rule_base = match['rule'].split('[')[0].replace('⚔️','')
    deviation_groups[rule_base].append(match)
# 输出 dev≥2 按规则链分组 → 看到底是哪个规则链出的问题
```

### 4. 单点修复的连锁反应检查

```python
# 每fix一条规则后，必须全量回测
# 常见连锁反应:
# - fd阈值改宽 → 误入相邻分支
# - 条件放宽 → 同级规则被跳过
# - r3_already_handled 缺漏 → motivation推高正确比分
```

### 5. 常见反模式

| 反模式 | 后果 | 正确做法 |
|:-------|:-----|:---------|
| 改fd阈值解决单场比赛 | 破坏5-6场其他比赛 | 加新规则分支，非改现有阈值 |
| 在赛后数据上加条件 | 下次拉取数据不同又失效 | 确认赛果，调整规则优先级 |
| 忽略r3_already_handled | R3 motivation推高所有非R3规则 | 加豁免 |
