## 铁律：回测必须包含基本盘

**用户铁律（2026-06-22）：回测时必须传入weather/temp/neutral参数。**

```python
# ❌ 错误：只传赔率
predict(h="阿根廷", g="奥地利", fh=1, fa=24, o1=1.46, o3=7.33, ..., rd=2)

# ✅ 正确：包含基本盘
predict(h="阿根廷", g="奥地利", fh=1, fa=24, o1=1.46, o3=7.33, ..., rd=2,
        temp=31, neutral=1, weather=7)
```

**原因：** predict()内置基本盘修正（温度≥30°C降λ×0.90，中立×0.95）。不传这些参数导致回测与实际预测结果不一致。

## 全量回溯流程（28场世界杯验证）

```
Step 1: 收集所有已完赛matchID（按销售日分组）
Step 2: 对每场拉取新浪API数据：
   ├─ footballMatchDetail → team1Position, team2Position, environment, isNeutral, round
   ├─ footballMatchOddsEuro → 53家初赔即赔, 升/降家数
   └─ footballMatchOddsAsia → 盘口变化, 升降盘家数
Step 3: 解析environment字段提取温度(°C)
Step 4: 传入predict()包含temp/neutral参数
Step 5: 对比预测vs实际，分三级记录：
   ├─ 🎯 精确命中 (h==ah && a==aa)
   ├─ ±1 偏差≤1球 (abs(h-ah)<=1 && abs(a-aa)<=1)
   └─ ❌ 偏差≥2球
Step 6: 发现系统偏差 → 分析根因 → 修复规则链 → 重新全量回放
Step 7: 回归检查：验证原有命中场次不退化
```
