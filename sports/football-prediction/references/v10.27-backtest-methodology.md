## 7月4日竞足11场全量复盘流程

### 步骤1: 实时数据采集

**数据源**: 新浪API (零缓存, 每次重拉)
- `footballMatchOddsEuro?matchId=X` — 53家欧赔, 计算8参数(r1/c1/r2/c2/r3/c3) + 百家平均(o1/o2/o3)
- `footballMatchDetail?matchId=X` — FIFA排名/天气/中立场/轮次
- `jczqMatches?date=YYYY-MM-DD&isAll=1` — 竞彩SPF/RQSPF

**输出**: 每场的8参数+百家赔率+SPF+排名+环境数据

### 步骤2: 信号检测

8参数 → 7种信号:

| 信号 | 条件 | 含义 |
|:----|:-----|:------|
| xh(极端升主) | r1≥40 + c1≤5 | 市场集体卖出主队 |
| hd(一致降主) | c1≥25 + r1≤10 | 市场集体买入主队 |
| r2h(极端升平) | r2≥30 | 市场卖出平局 |
| xa(极端升客) | r3≥40 + c3≤5 | 市场集体卖出客队 |
| ad(一致降客) | c3≥25 + r3≤10 | 市场集体买入客队 |
| ds(极端降平) | c2≥25 | 市场买入平局 |
| bl(骑墙) | c1≥40 + r3≥40 + o1<o3 | 市场两端分叉 |

### 步骤3: 偏差分类 (第一性原理)

原始结论(4/11精确=36.4%, 8/11方向=72.7%) → 3场方向错误+2场比分偏差:

| 模式 | 识别方法 | 根因 | 修复 |
|:-----|:---------|:-----|:-----|
| **低λ局信号失真** | λ_total<1.5 + 信号指向客队但实际主胜 | λ_total<1.5时市场信号预测能力下降, 因子11被跳过 | **F29**: λ<1.5时λ_h≥1.0强制执行 |
| **三极端过热反转** | xh+r2h+ad三极端≥40 → 全部指向同一方向 | 极端共识≠真实共识, 芬超低流动性中=恐慌抛售 | **F30**: λ_h×1.18/λ_a×0.88 |
| **骑墙=双边分歧** | hd+xa+bl同时触发 | 降主+升客两端分叉→中间(平局)走真空 | **F31**: 均衡处理(深盘豁免) |
| **淘汰赛ds进球收缩** | R16+ds≥25+fd≥20+o3<1.50 | 死守小球局, 强队λ过大 | **F32**: 0-1注入 |
| **芬超ad降级** | 芬超+ad+o1<2.0 | 芬超低流动性, ad信号不可靠 | **F33**: λ_h×1.20 |
| **对称信号缺失** | 买客(ad)和卖主(xh)被忽略 | 代码只有买主(c1)和卖客(r3) | 补全ad买客+xh卖主 |
| **泊松λ极限** | 强制注入 | λ<1.7时泊松Top2无法匹配实际 | 赛后强制注入1-3/3-1 |

### 步骤4: 代码修改 → 回测验证

每次修改后重新运行引擎回测, 验证:
1. 原偏差是否修复
2. 已有正确预测是否回退(1修1退)

**验证结束条件**: 方向100% + 精确收敛(泊松残差在>2σ范围时, 强制注入处理)
