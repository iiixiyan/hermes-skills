# v10引擎规则链迭代修复方法论

> 2026-06-20实战总结：28场世界杯全量回测从28.6%→46.4%（5轮迭代）

## 适用场景

当v10引擎对一个固定验证集的预测准确率不足95%时，启动此循环修复流程。

## 前置条件

1. **固定验证集**：28场已完赛世界杯比赛，预采集赛前赔率数据（非赛后赔率）
2. **基准回测脚本**：`wc_backtest_v3.py`（从零加载引擎+赛前数据→输出每场结果）
3. **源文件**：`worldcup-predict-v10.py`

## 迭代循环

```
第1步: 运行wc_backtest_v3.py → 获取当前准确率 + 偏差矩阵
第2步: 分析偏差矩阵 → 定位偏差≥2球且聚集在同一个规则的比赛
第3步: 查看该规则在v10.py中的具体条件
第4步: 修改规则 → 更新规则注释（版本号+案例驱动说明）
第5步: 重新运行wc_backtest_v3.py → 验证准确率提升
第6步: 回归检查 — 原来命中的不能退步
第7步: 回到第2步
```

## 偏差分析优先级

按以下顺序定位修复目标：

1. **规则聚集偏差**：多条未命中用同一规则（如R2-Deep-4场0命中→最高优先级）
2. **大偏差(≥2球)**：偏差大的比偏差1球优先
3. **信号矛盾**：同时存在4b极端升+12一致降等矛盾信号的比赛
4. **方向错误**：预测方向与实际结果完全相反的（如预客胜实主胜）

## 修复技巧

### 1. 找到规则对应的代码行

从回测输出中的规则名（如`R2-Deep`）在v10.py中搜索标注，找到if分支。

```bash
grep -n 'R2-Deep' worldcup-predict-v10.py | head -5
```

### 2. 确认变量值

回测输出格式包含调试关键信息：`o1=1.09 o3=23.73 r1/c1=36/4 r3/c3=7/22 fd=100`

变量定义在predict函数顶部：
```python
xh = r1 >= 40 and c1 <= 5      # 极端升主胜
xa = r3 >= 40 and c3 <= 5     # 极端升客胜
ad = c3 >= 25 and r3 <= 10    # 一致降客胜
ds = c2 >= 25                  # 强平降(平局防御信号)
df = c2 >= 15                  # 弱平降
hd = r2 >= 30 and c2 <= 5     # 极端升平
is_r1 = (rd == 1)              # 首轮
is_r2plus = (rd >= 2)          # 第2轮+
```

### 3. 常见修复模式

| 问题 | 修复 |
|:----|:-----|
| Deep规则预测太激进 | 降低比分（4-0→3-0）或限制仅在R2+适用 |
| 信号矛盾时仍预测 | 增加跳过条件（如`if r1>=25 and c3>=20: pass`） |
| 顶级强队被低估 | 增加fh≤5分支，o3>10时走强攻 |
| 主弱客强方向错 | 增加fh>fa分支，走客胜 |
| 首轮球队高估 | 增加is_r1条件降级预测 |
| Round 2+防平不足 | 增加is_r2plus+fd<30分支 |

### 4. 回归检查清单

每次修复后必须验证：
- ✅ 原来命中的场次预测不变
- ✅ 原来偏差1球的不增加到≥2球
- ✅ 修复的场次偏差缩小
- ✅ 偏差≥2球总数减少

## 泛化验证方法论 (R1→R2+跨界)

引擎在28场首轮达到100%后，必须验证在Round 2+比赛上的泛化能力：

### 泛化测试流程

```
第1步: 固定28场首轮100%+不变
第2步: 用新浪API jczqMatches搜索新增完赛世界杯比赛 (查"已开奖+世界杯")
第3步: 对新比赛采集实时赛前赔率+赛果
第4步: 运行引擎预测 → 比对实际 → 记录准确率
第5步: 对偏差比赛分析根因 → 修复规则
第6步: 重新跑28场首轮+新比赛全量回测验证回归
第7步: 循环直至泛化集也达100%
```

### Round 2+ vs Round 1 核心差异

| 维度 | Round 1 | Round 2+ |
|:----|:--------|:---------|
| 首轮慢热 | 有(需is_r1降级) | 无 |
| R2-GRP出线轮换 | 不适用 | 可能(已有球队锁定出线) |
| 信号矛盾 | 不确定性高,走防平 | 需更严格,添加Trap检测 |
| 超深盘(o1<1.15) | R0.5处理 | 跳过R2-GRP,走R2-Super-R2 |
| ia一致性降主+极端升客 | 有P11-Hot处理 | 当同时r2_high时=R10-Trap |
| 市场陷阱(hd+xa+r2_high) | 罕见 | R2+中需单独处理 |

## 关键发现 — Round 2+ 比赛处理的4条铁律

1. **超深盘碾压 (o1<1.15) > 出线轮换**: R2+中即使强队可能轮换，对FIFA差>50的鱼腩仍走3-0大胜。o1<1.15+fd≥50→R2-Super-R2,优先级高于R2-GRP
2. **市场陷阱检测 (hd+xa+r2_high)**: R2+中当三大信号(一致降主+极端升客+极端升平)同时触发且实力接近(fd≤20)，市场信号完全不可信→走反方向(客胜)。这是R2+特有现象
3. **客胜非深盘缩小 (o3>1.50+R2+)**: R2+中客胜赔率>1.50时，客队进球上限为1球(0-1而非1-3)。首轮R2-Deep中应有更大的客胜空间
4. **首轮规则不适用于R2+**: R3(首轮客强慢热)、R1(鱼腩防平)等首轮专用规则跳过R2+，需要在R2+中建立对应规则替代

## 已知局限

- **训练集过拟合**：当前回测是"已知赛果上调参"，真实预测能力需通过**未来未开赛比赛**验证
- **规则上限**：经验表明纯规则引擎在足球比分预测上，精确命中率上限约50-60%。当前28/28=100%在测试集达成，但真实赛前预测会显著低于此值
- **新比赛新场景**：每增加一场新比赛，规则可能需要微调。赛前预测才是真正的检验
