## 复盘经验（2026-05-17 复盘积累）

### ⚠️ 复盘核心铁律（2026-05-24 用户纠正）

**所有复盘必须以比分为唯一核心。** 盘口信号分析、基本面验证、模型信号复盘不是并列检查项——它们围绕比分展开：

```
比分是圆心 ← 盘口信号（解释为什么这个比分没命中）
           ← 基本面验证（什么因素导致比分偏差）
           ← 模型信号（定位比分偏差的源头）
```

**输出优先级（不可违反）**：
1. **比分命中率**（精确+方向+偏差分布）— 必须放在最前面
2. **比分偏差归因**（每场没命中的具体原因：λ偏差？因子漏触发？数据错？）
3. **改进方向**（基于归因，下次怎么调）

联赛分区分析、盘口信号统计是**次要内容**，放在比分分析之后。每个盘口/基本面分析结束时必须回到「→ 这对比分预测意味着什么」。

**前沿方法雷达**：bjdc-prediction 技能已集成 ArXiv 2024-2026 最新论文成果（市场校准、双变量泊松、Sarmanov 族等），详见 `sports/bjdc-prediction/references/research-papers.md`。竞足相关方法（尤其是市场校准、Dixon-Coles τ 扩展）可跨技能复用。

以下洞察来源于2026-05-17（周日）33场竞足比赛的复盘分析。**每次实际分析时必须查阅这些经验，特别是赛季末段。**
这些因子已迁移到下方「复盘验证因子库」中统一维护，并按因子1-9编号。

### 2026-05-25 竞足复盘补充

**数据：** 9场（挪超6/瑞超2/德甲1），精确命中6/9=66.7%，方向错3/9

**核心发现：**
1. **平局预测极强** — 5场1-1精确命中，模型对均衡局平局判断力出色
2. **挪超主胜被严重低估** — 3场方向错全在挪超，全部预测客队不败实际主胜
3. **保守偏客系统性偏差** — 当挪超主队排名不低于客队5名时不应默认走客

**改进建议：**
- 挪超主场系数从+10~15%上调至+15~20%
- 挪超主队排名不低客队5名时优先主队不败
- 新增「均衡局+多场不胜→平局」触发条件

**⚠️ 复盘前必须执行第0步：验证预测来源**

在读取DB预测数据前，先通过 `session_search` 查询前一日的cron预测任务（`cd0456f2c803`）会话摘要。若摘要显示"对话截断"或"预测未完成"，则DB中数据可能为不完整版本——立即向用户确认，不可直接使用。案例：2026-05-24复盘误用17:00截断cron的26条预测。

**复盘核心原则：必须按联赛分组分析，不得混在一起统计。** 不同联赛的盘口规律、主场系数、冷门率差异极大，混在一起分析会导致规律被稀释。详见 `references/league-review-templates.md`。

**数据获取**：
- 使用竞足开奖结果页面 **`https://kt.59itou.com/jingcai/prize/`** 获取已完成比赛（⚠️ `www.59itou.com` 域名下该页面可能因证书错误或404无法访问，优先使用 `kt.59itou.com` 子域名）
- 亦可尝试: `https://www.59itou.com/jingcai/prize/`（HTTPS存在证书问题，如明确ERR_CERT则改用kt子域名）
- 默认显示"昨日"最新比赛日数据，点击日期标签（今日/昨日/5-18/5-17等）可切换比赛日
- 页面底部有更多历史日期标签
- 使用 `document.querySelectorAll('[class*="item"]')` 定位比赛条目元素，**每个条目的 `id` 属性即为 matchID**（如 `id="2671925"`）
- 每条item内嵌"析"文本（`a`标签）可直接跳转到详情页
- 各SP字段的顺序（每个prizeitem内）：联赛名 → 场次(周二001) → 主队名 → 比分 → 客队名 → "析" → 胜平负SP → 让球SP → 比分SP → 总进球SP → 半全场SP
- 注意：prize page 只显示**中奖SP赔率**（即赛后真实赔率），不是全量赛前赔率
- 完整的赛前赔率（胜/平/负三项）需从竞足列表页详情页的**欧指Tab中的百家平均即赔/初赔**获取
- ⚠️ **已完成比赛（开奖结果页面）的亚指Tab不可用**：已结束的比赛在详情页亚指Tab会显示"参数错误"，并非网络问题。这是页面JS渲染限制，无法获取已完成比赛的盘口水位数据。欧指Tab和战绩Tab数据正常可获取。

**联赛分组分析步骤**：
1. 获取所有比赛数据后，按联赛名称分组归类
2. 按联赛分组体系分类：五大联赛各自独立、亚洲/北欧/南美分别归类、洲际赛事单独、其他小联赛按场次决定深度
3. 每个联赛独立输出：赛果概览表 → 联赛规律分析（主胜率、冷门率）→ 盘口信号统计 → 联赛参数更新建议
4. ≥3场的联赛用完整模板，<3场的用简要模板
5. 最后输出汇总表展示所有联赛的关键发现
6. 联赛参数需连续3天稳定趋势才更新到SKILL.md，单日异常不纳入
