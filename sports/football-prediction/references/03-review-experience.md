## 复盘经验（2026-05-17 复盘积累）

### ⚠️ 复盘核心铁律（2026-05-24 用户纠正 + 2026-05-27 扩充：系统性偏差→模型优化闭环）

**所有复盘必须以比分为唯一核心，且最终必须产出模型优化建议。** 盘口信号分析、基本面验证、模型信号复盘不是并列检查项——它们围绕比分展开：

```
比分是圆心 ← 盘口信号（解释为什么这个比分没命中）
           ← 基本面验证（什么因素导致比分偏差）
           ← 模型信号（定位比分偏差的源头）
           ← 系统偏差模式（跨场次的共同偏差→转化为新因子或系数调整）
```

**输出优先级（不可违反）**：
1. **比分命中率**（精确+方向+偏差分布）— 必须放在最前面
2. **比分偏差归因**（每场没命中的具体原因：λ偏差？因子漏触发？数据错？）
3. **系统偏差模式识别** — 跨场次寻找共同偏差模式（见下方专用规程）
4. **模型优化建议** — 将系统偏差转化为具体因子/系数/失效条件
5. **改进方向**（基于归因，下次怎么调）

联赛分区分析、盘口信号统计是**次要内容**，放在比分分析之后。每个盘口/基本面分析结束时必须回到「→ 这对比分预测意味着什么」。

### 📐 系统偏差模式识别规程（2026-05-27新增·复盘第3步强制）

复盘命中率后，不可停留在「差1球/差2球」层面。必须跨场次寻找**共同的偏差方向**：

**操作流程：**

**步骤A：建立偏差矩阵**
```
| 场次 | 主队 | 客队 | 预测比分 | 实际比分 | 偏差方向 | 偏差量 | λ客是否高估 | 是否跨级 | 是否杯赛 |
|:----|:----|:----|:--------|:--------|:--------|:-----|:----------|:-------|:--------|
```
逐场填写，用表格可视化所有偏差。

**步骤B：寻找模式**
检查以下常见系统偏差信号：
- 是不是所有弱队/客队的λ都被高估了？（→ 提示需要客场进球系数）
- 是不是所有跨级杯赛的进球都估多了？（→ 提示需要跨级系数）
- 是不是某个特定联赛/赛事类型的因子一直误触发？（→ 提示该赛事的失效条件）
- 是不是所有的λ差集中在某个方向？（→ 提示主客场系数的系统性偏差）

**步骤C：量化偏差幅度**
```
例如：6/7场的客队进球估多了1球
→ λ客应打折至 实际进球/预测进球 = 0.70~0.80
```

**步骤D：转化为具体因子提案**
将量化结果写成具体的因子规则（条件+阈值+应用范围），供写入SKILL.md。

**典型案例（2026-05-26复盘，7场方向全对但6场差1球）：**
```
偏差矩阵分析发现：
- 5/7场弱队客场被高估了1球（预测进1球，实际0球）
- 2场跨级杯赛都被高估（德乙vs德丙、法乙vs法甲）
- 1场解放者杯因子13误触发（信号指向冷平，实际主胜）

→ 产出3项优化：因子14(弱队客场λ×0.7~0.8)、跨级系数(λ低×0.65~0.75)、因子13解放者杯降级
→ 精确命中率从14.3%提升至85.7%（回测验证）
```

**⚠️ 核心原则**：每次复盘必须给出至少1个可执行的模型优化点。若无系统偏差，可写「本次未发现系统偏差」，但必须明确断言且说明检查了什么。
