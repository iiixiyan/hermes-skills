## 数据流（3条并行线）

```
线A: 赛果采集（okooo，多日期循环）
for date in 全月日期:
  请求 okooo SportteryScore
  过滤非世界杯场次
  整理 {场次号, 主客队, 比分, SP}

线B: 欧指数据（新浪API）
for mid in 所有联赛matchID:
  请求 footballMatchOddsEuro → {百家平均, 指数变化}
  请求 footballMatchOddsAsia → {升降盘, 高水/低水}

线C: 预测文件
  读取 ~/.hermes/predictions/jingzu/YYYY-MM-DD.md
  或 cron output → {λ值, 比分, 方向, 因子检查}
```

## matchID 匹配（最关键脏活）

新浪API返回的matchID与预测场次号完全不同。通过SPF值+让球数双重确认关联：

```
预测: 3202 SPF 1.46/4.10/5.00 RQ=-1
新浪: 3635511 SPF 1.46/4.10/5.00 RQ=-1 → 匹配 ✅

预测: 3204 SPF 2.25/3.20/2.69 RQ=-1
新浪: 3635514 SPF 2.25/3.21/2.68 RQ=-1 → 匹配 ✅
```

**技巧**：让球数(RQ的+1/-1)是强区分符。SPF接近+让球一致=确认。

## 尾事件判定标准

```python
P = Poisson(实际主_进球, λ₁) × Poisson(实际客_进球, λ₂)
P < 5% → 尾事件，不计入系统性偏差
```
