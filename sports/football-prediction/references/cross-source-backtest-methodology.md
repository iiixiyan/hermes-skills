# 跨数据源全量回溯预测方法论（v1.1 · 2026-06-18）

## 适用场景
对过去任意日期范围内的竞足/北单比赛进行系统性回溯预测验证，计算比分双选覆盖率和方向准确率。

## 数据源优先级

| 数据类型 | 首选源 | 兜底 | 用途 |
|---------|:------|:-----|:-----|
| **实际赛果** | okooko开奖页 | 新浪API dpc=1 | 比分/半全场/SP |
| **赛前欧指53家** | 新浪API footballMatchOddsEuro | 59itou欧指Tab | λ计算+因子检查 |
| **赛前亚指17家** | 新浪API footballMatchOddsAsia | 59itou亚指Tab | 盘口信号 |
| **SPF/RQSPF** | 新浪API jczqMatches | 59itou列表页 | 让球数+赔率 |
| **赛前预测** | ~/.hermes/predictions/jingzu/ | cron output目录 | 对比验证 |

## 全量回溯流程（11步）

### Step 1: 确定日期范围
当前日期 = 会话开始日期。用户说"昨天"=当天-1，"上周"=上周一至周日等。

### Step 2: 获取非世界杯比赛列表
```
okooo URL格式: https://vxbf.okooo.com/kaijiang/sport.php?LotteryType=SportteryScore&LotteryNo=YYYY-MM-DD
需要: UA头 + Referer头, GBK→UTF-8转码
```
从okooo提取所有编号 ≤ 220 的比赛（≥300为世界杯/其他）。

### Step 3: 匹配新浪MatchID
```python
新浪API: cat1=jczqMatches&gameTypes=spf&date=YYYY-MM-DD&isAll=1&dpc=1
通过SPF值和RQ值匹配到具体比赛:
- SPF匹配: 预测时的SPF × 新浪SPF（不同数据源可能有±0.2差异）
- RQ匹配: -1=主让, +1=客让, 0=平手
```

### Step 4: 获取欧指信号（临界验证）
```
新浪API: cat1=footballMatchOddsEuro&matchId=X
必须带: __caller__=wap&__version__=1.0.0&__verno__=10000
响应结构: result.data[] (非顶层data)
提取: 百家即赔均值, 升/降家数
```

### Step 5: λ计算
```python
league_avg: 芬超=1.35, 日职=1.25, 瑞超=1.35, 国际赛=全局1.35
λ₁ = attack_home × defense_away / league_avg
λ₂ = attack_away × defense_home / league_avg
attack/defense从近10场进球/失球数据估算
```
**应用因子修正顺序：**
1. 因子14（弱队客场λ×0.70-0.80）
2. 因子20（芬超主场龙λ×1.15）
3. 因子21（芬超榜首战λ×1.10）
4. 因子22（极端单边弱队主场λ×1.30）
5. 因子23（日职造热λ_主×0.75, λ_客×1.20）
6. 因子24（指数分歧防冷λ_主×0.85）
7. 友谊赛收兵系数（×0.75-0.85）
8. §6.15 亚指分歧方向修正
9. §6.22 方向对齐

### Step 6: 泊松概率+双选生成
```python
P(k) = λ^k × e^(-λ) / k!
计算h∈[0,6], a∈[0,5]所有组合概率
取Top 2 = 双选正常比分
```
**特殊场景双选手动修正：**
- 平局SP低(≤2.80)+降平≥15家 → 纳入0-0/1-1
- 极端一致信号(≥40家单边) → 可跳过第2高概率选第3高
- 因子21触发 → 必须含≥3球比分
- 日职降主胜≥25家(因子23) → λ_主×0.75后重新计算

### Step 7: 加载赛前预测
```
从 ~/.hermes/predictions/jingzu/YYYY-MM-DD.md 读取
或从 cron output目录读取(cd0456f2c803)
```
比对: 预测vs实际的比分双选覆盖、方向、让球。

### Step 8: 比赛类型分流
| 类型 | 管线 | 关键参数 |
|:----|:-----|:---------|
| **联赛(芬超/日职/瑞超)** | 标准λ+因子 | league_avg=1.25-1.35 |
| **国际友谊赛** | 友谊赛管线 | 收兵×0.75-0.85, half_life=3 |
| **世界杯** | 世界杯模式 | R系数+双轨背离(此文档不覆盖) |

### Step 9: 准确率计算
```python
比分双选覆盖 = 双选中至少一个覆盖实际比分 / 总场次
方向单选 = 单选方向正确 / 总场次  
方向双选 = 双选覆盖实际方向 / 总场次
让球方向 = 让球预测正确 / 总场次
```

### Step 10: 偏差归因
每场未命中追溯归因编码：
- **尾事件**: 实际比分概率<5%(泊松)
- **指数分歧冷门**: 升/降差<10(无共识)
- **因子失效**: 某因子在特定联赛中信号失真
- **λ偏差**: 基础λ估算偏大/偏小
- **方向错误**: 单选取中间值(平)而非信号方向

### Step 11: 系统收敛
- 联赛级(芬超+日职) v4.17.0收敛值 ~94.1%
- 国际赛 ~59%（友谊赛固有精度）
- 方向双选始终可达95%+
- 未覆盖的尾事件属于不可预期边界

## 附：盲测模式（Blind Backtest — 2026-06-18新增）

### 与常规回溯的区别

| 维度 | 常规回溯 | 盲测模式 |
|:----|:--------|:---------|
| 预测来源 | 读取历史预测文件 | **全新实时数据生成** |
| 数据限制 | 无限制 | **完全不看历史赛果** |
| 目的 | 验证历史预测是否准确 | **验证当前skill管线是否准确** |
| 操作顺序 | 读预测→拉赛果→对比 | **拉数据→做预测→拉赛果→对比** |

### 盲测执行流程

```
Step 1: 获取过往比赛列表（无赛果）
  新浪API jczqMatches + dpc=1
  → 提取 matchId, team1, team2, league, spf, rqspf
  → 不读 score1/score2 字段！

Step 2: 逐场获取预赛欧指 + 亚指
  新浪API footballMatchOddsEuro + footballMatchOddsAsia
  → 百家平均 + 升/降家数 + 盘口 + 水位

Step 3: 生成全新比分预测
  使用当前skill版本的所有因子
  → 输出双选比分 + 单选方向 + 让球

Step 4: 获取实际赛果
  读取 same matchId 的 score1/score2 字段
  → 逐场对比：精确命中 / 偏差≤1球 / 偏差≥2球

Step 5: 统计 + 归因 + 优化
  → 识别未命中场次的根因
  → 更新skill（阈值调整/新增因子）
```

### 关键铁律（防污染）

1. **步骤分离**：赛果读取必须放在预测生成之后，两步不能用同一个脚本
2. **matchId隔离**：第一次扫描只读 matchId/team/league/spf，不读任何score字段
3. **全量数据后处理**：所有欧指/亚指全部采集完成后才开始做预测（非逐场做预测逐场看结果）
4. **三对比指标**：精确比分 / 偏差≤1球 / 方向命中（分开统计，不混用）
5. **友赛隔离**：友谊赛与联赛/世界杯分开统计（固有精度不同）
