# 2026-07-07 竞足复盘（F41新增 + 数据源选择教训）

## 复盘数据源铁律（2026-07-08经验）

**复盘时的预测数据必须来源于会话消息(session_search)中实际发出的预测，而非cron输出文件。**

原因：
- cron输出文件(`~/.hermes/cron/output/`) 头部包含大量prompt模板内容
- 实际发出的预测可能在prompt导入后经agent重写修正
- 仅cron输出文件末尾部分(最后几百行)是预测内容，但可能已被agent在交付前修改

**正确做法**：
1. session_search(query="竞足 预测 已发出") 找到已发送的预测记录
2. 从assistant消息中提取实际预测内容（通常在消息中段的数据表格）
3. cron输出文件仅作为交叉验证，不可作为唯一数据源

## F41 — 极端百家反转豁免造热处理

**发现来源**：维京古(雷克维京)1-0杰尔(欧冠资格赛)

### 原始信号
| 指标 | 初赔→即赔 | 变化 | 8参数 |
|:----|:----------|:---:|:-----|
| 主胜 | 4.194→2.388 | **-43.1%** ↓↓ | c1=40(hd) |
| 客胜 | 1.934→2.660 | **+37.6%** ↑↑ | r3=39(xa接近) |
| 平局 | 4.010→3.444 | -14.1%↓ | c2=30(ds) |

### 引擎行为
引擎输出 R6-BL造热陷阱 → 1-1/1-2 ❌

### 实际赛果
**1-0 主胜**

### 第一性原理分析
1. 主胜-43.1%的幅度=史上最强级别的赔率变动之一,远超8参数家数信号的可信度
2. 客胜从1.934→2.660(+37.6%)说明市场完全否定了开盘时的客队热门预期
3. 冰岛主场因素：人工草皮、5°C低温多雨、客队杰尔从匈牙利长途飞行
4. **比例反转(±35%+)才是第一性信号**，8参数家数(40家降主)只是辅助验证

### 触发阈值
- 百家平均主胜降幅 ≥ 35%
- 且 百家平均客胜升幅 ≥ 30%
- 同时触发 → 豁免R6-BL/R10-Trap等造热处理，走主队方向

### 引擎实现位置
在R6-BL/R10-Trap分支前插入：

```python

# 触发: 主胜降幅≥35% + 客胜升幅≥30%
if o1_change <= -0.35 and o3_change >= 0.30:
    # 真实市场重评估,走主队方向
    return home_direction_score
```
