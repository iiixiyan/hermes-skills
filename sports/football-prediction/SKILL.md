# 竞彩足球比分预测（v4.6.0 重构版）

> 采用 Progressive Disclosure 架构。本文件仅放触发条件、核心流程、引用索引。详细方法论见 references/ 目录。

## ⚠️ 技能边界

**此技能仅用于竞彩足球（竞足）分析，不适用于北单（北京单场）。**

| 维度 | 竞足 → football-prediction | 北单 → bjdc-prediction |
|:----|:--------------------------|:-----------------------|
| 输出内容 | 比分预测（λ计算、xG模型、双轨比分） | 北单选项（让胜/让平/让负） |
| 核心模型 | 泊松分布λ加权、状态权重、攻防结构 | 盘口技术手册、诱/阻判定、信号速查 |
| 输出格式 | 〖竞足-XXXX〗 + 正常比分/异常比分 | 📊 北单推荐：让胜/让平/让负 |

**混淆禁止**：
- ❌ 北单分析中不得输出比分预测
- ❌ 北单分析不得使用football-prediction的λ公式和xG模型
- ❌ 北单分析不得使用〖竞足-XXXX〗格式头

## 二、核心流程（5步压缩版 v4.6.0）

### 第1步：数据采集
- 竞足列表 → `kt.59itou.com/627/jingcai/` → 北单列表 `883/danchang/` 提取 matchID
- 逐场详情页（lotteryId=90）6Tab innerText实时采集
- **数据强制实时**：禁止引用任何历史缓存数据（参见 ⚠️ 数据强制实时获取节）
- **采集失败 fallback**：手动输入关键数据（三赔+让球数+近期战绩），Skill仍可运行

### 第2步：λ锚定
```python
# λ_stats 计算（内联核心公式）
λ_stats_主 = (近3场均进球×0.4 + 近6场均进球×0.3 + 赛季场均×0.3) × 主客场系数
λ_stats_客 = (近3场均进球×0.4 + 近6场均进球×0.3 + 赛季场均×0.3)

# 赛程密度修正（若近3场跨度>30天→降低近3权重至0.25, 赛季提至0.45）
if 近3场跨度 > 30: λ_stats = 近3×0.25 + 近6×0.3 + 赛季×0.45

# 弱队客场λ修正（因子14触发时）
if 客排名>主5名 or 跨级杯赛: λ_stats_客 ×= 0.70~0.80

# λ_market = 网格校准（argmin|P_model - P_market|²）
# λ_final = 0.5×λ_market + 0.3×λ_stats + 0.2×λ_transitive（桥接法）
```

> 详细λ计算与桥接法：`references/analysis-framework.md`, `references/05-bridge-method.md`

### 第3步：因子速查
遍历14因子，逐因子核对触发状态。核心因子速查表直接内联：

| 优先级 | 因子 | 触发条件 | 执行动作 |
|:--|:--|:--|:--|
| **P0** | 4b 一致升赔 | ≥20升/≤2降 | **→强制检查失效条件(杯赛决赛/中立场/换帅)** → 异常含反方向 |
| **P0** | 2 赛季末战意 | 末5轮+战意差 | 异常权重+2级 |
| **P0** | 14 弱队客场λ | 客排名>主5名/跨级杯赛 | λ客×0.70~0.80 |
| **P1** | 12 一致降赔 | ≥20降/≤2升+基本面异常 | 造热预警 |
| **P1** | 13 双边分歧 | 一端≥20升+另端≥20降 | 平局+35%（⚠️解放者杯降级） |
| **P1** | 4b-EXT | ≥20升/≤5降+幅度≥3% | 标记 |

> 完整14因子含案例库：`references/01-factor-table.md`

### 第4步：盘口校验
- **让球独立判定**：用亚指盘口水位数据独立推导让球方向（严禁从比分反推）
  - 升盘X家/降盘Y家，主队高水Z家/低水W家 → 倾向穿盘/不穿盘
- **亚指信号摘要**（输出时附 ≤25字）：`亚指：升X/降Y，主水高Z家 → 倾向不穿 → 让平`
- **异常比分协同校验**：若异常比分为大比分(4-1)但让球方向=让平 → 触发一致性警告

### 第5步：输出
```python
🎯 正常比分：[比分A]/[比分B]       # 双选，λ模型最可能的2个
🎯 胜平负：[胜/平/负]              # 独立推导，严禁从比分反推
🎯 让球：[让胜/让平/让负]           # 独立推导，附盘口信号摘要

🎯 异常比分：[比分A]/[比分B] 🏷️ [A1/B1等]  # 双选含标签+H2H印证
⭐ 信心：[★~★★★★★]
```

> 详细异常路径：`references/02-abnormal-score-paths.md`
> 5串1模板：`templates/5x1-recommendation.md`

### 第7步：串关筛选（按需触发，cron任务必输）
当用户要求串关/合买/稳单时。详见：
- `references/07-prediction-tracker.md` — 凯利配比、投注额建议
- `references/sp-odds-alignment.md` — 场次筛选、双选有效赔率、输出格式

## 四、输出铁规

**输出格式（v4.6.0含推导校验）：**
```
〖竞足-XXXX〗 [主队] VS [客队]（[联赛]）⚽ 让球[±N]

📊 数据：近X场 [主队]X胜X平X负 进X失X | [客队]X胜X平X负 进X失X

🔍 λ锚定：λ主=X.XX λ客=X.XX 差=X.XX（近3×0.4+近6×0.3+赛季×0.3）
📌 因子：4b[触发/X升Y降] 12[触发/未] 13[触发/未] 14[触发/未]
⏱️ 数据时效：[采集时间HH:MM → 距今Xh]

🎯 正常比分：[比分A]/[比分B]
🎯 胜平负：[胜/平/负]  ← 推导依据：[λ差/因子4b/盘口]
🎯 让球：[让胜/让平/让负]  ← 推导依据：[升X家/降Y家/高水Z家 → 倾向]

🎯 异常比分：[比分A]/[比分B] 🏷️ [A1/B1] [✅H2H历史印证]
⭐ 信心：★★★☆☆
```

### 铁律
- ✅ **正常比分**：必须双选（2个比分），λ模型选最可能的2个覆盖
- ✅ **胜平负**：模型综合λ差+因子+市场信号**独立判定**，**严禁从比分反推**
  - 例：因子4b一致升主胜→即使预测2-1，胜平负也可能=负（冷门预警）
  - 例：λ差<0.30+因子12一致降赔→即使预测主胜，胜平负也可能=平
- ✅ **让球**：模型综合盘口水位+欧指信号**独立判定**，**严禁从比分+让球数反推**
  - 例：让球-1+比分2-0，但亚指水位不支持穿盘→让平而非让胜
  - 判断依据：亚指升盘公司数/降盘公司数、主队水位高水/低水公司数、盘口走势
- ✅ **异常比分**：必须双选，须标注类型标签（🔴A1-A4 / 🟠B1-B4）+ 触发条件
- ✅ **所有数据必须实时采集**：禁止使用任何之前会话中缓存/保存的比赛数据（innerText快照、欧指变化、战绩等），终选前必须重新拉取欧指数据
- ✅ 每场必须输出让球胜平负结果（让球数从59itou列表页(±N)提取）
- ❌ 禁止用比分反推胜平负或让球（必须基于因子独立判断）
- ❌ 禁止爆冷和大比分混在同一组
- ✅ 可用 `scripts/extract_h2h_scores.py` 从innerText提取并标注H2H常见比分
- ❌ 禁止"仅供参考"等免责套话
- ❌ 禁止跳过λ计算（近3/近6必须展开，赛季可简写）
- ❌ 禁止使用简化脚本/模板化输出
- ⚠️ 终选前必须重新拉取欧指数据（因子4b会漂移）
- ⚠️ **欧指Tab指数变化段缺失时，必须重新加载详情页**（browser_navigate+等待2s后再取）。连续2次取空→标记"欧指数据不完整"并使用百家平均直接判断升降方向（初赔vs即赔差价≥5%=有效升降）
- ⚠️ **因子4b执行前强制检查失效条件**：杯赛决赛❌、中立场地❌、近3场换帅❌。任一命中→4b降级为标记仅参考（不从P0执行）
- ⚠️ **价值回归识别规则**（v4.5.0新增）：当主胜≥20升/≤2降触发4b，但同时满足"主队近10场胜率≥70%"且"主胜初赔<2.0"且"平局降≥20家+客胜降≥20家"→判定为"市场价值回归"（初赔过低的市场修正），4b降级为P1参考，异常比分主队不胜方向可选不强制

### 串关输出铁规
- ❌ 禁止展开λ加权计算
- ❌ 禁止输出完整单场分析格式
- ❌ **禁止5串1中出现重复场次**（5场必须来自5场不同比赛）
- ✅ 用表格呈现场次+比分+信号+入选理由
- ✅ **风险等级列强制**：表格中增设 风险 列，🟢双模一致 / 🟡单模强信号 / 🔴高赔博单
- ✅ **风险梯度强制**：5串1至少1场🟢，至多1场🔴
- ✅ 必须附剔除场次及理由
