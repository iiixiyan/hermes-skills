# Hermes Agent 教程系列规划

## 系列概况

| 项目 | 内容 |
|------|------|
| 目标读者 | 纯新手（不假设有 AI Agent 使用经验） |
| 篇数 | 8 篇 |
| 发布节奏 | 一周两篇 |
| 写作风格 | 花叔风格 — 口语化、短句、数字驱动、融入真实体验 |
| 系列定位 | 从 0 到 1 掌握 Hermes Agent，每篇独立可读，整体递进 |

## 篇章规划

```
第一篇  ─ 入门篇    吸引安装，5分钟跑起来
第二篇  ─ 配置篇    换模型、调工具、配环境
第三篇  ─ Skills篇  Hermes最核心的特色
第四篇  ─ 日常篇    高频用好 slash 命令
第五篇  ─ 自动化篇  Cron + Webhook 解放双手
第六篇  ─ 多Agent篇 多实例协作与蜂群模式
第七篇  ─ 网关篇    接Telegram/微信/钉钉
第八篇  ─ 实战篇    完整项目：从0到发布
```

## 各篇详细规划

### 第一篇：👋 入门篇
**《5 分钟上手 Hermes Agent — 开源 AI 编程助手的正确打开方式》**

| 项目 | 内容 |
|------|------|
| 核心卖点 | 一句话安装、界面长什么样、能做什么 |
| 实操内容 | `curl -fsSL ... \| bash` → `hermes` → 问第一个问题 → 看它干活 |
| 信息增量 | Hermes vs Claude Code vs Codex 的区别？Hermes 的 3 个独特优势：Skills / Memory / Gateway |
| 写作风格 | 上手体验式，真实截图 |
| 字数 | ~2500字 |

### 第二篇：⚙️ 配置篇
**《玩转 Hermes 配置：换模型、配 Provider、调工具集》**

| 项目 | 内容 |
|------|------|
| 核心卖点 | Hermes 可以用 20+ 家模型，选对省一半钱 |
| 实操内容 | `hermes model` 切换 DeepSeek/Claude/GPT → `hermes tools` 开关工具 → 配置 fallback 链路防止 429 |
| 信息增量 | 20 个 Provider 怎么选？Fallback 链路的「同模型不同端点」技巧 |
| 写作风格 | 教程型，每步一个命令 |
| 字数 | ~3000字 |

### 第三篇：🧠 Skills 篇（核心）
**《Hermes 最独特的武器：Skills 技能系统从入门到精通》**

| 项目 | 内容 |
|------|------|
| 核心卖点 | 这是 Claude Code 和 Codex 都没有的能力——让 Agent 自己学会怎么干活 |
| 实操内容 | `skill_view()` 加载 → `skill_manage()` 创建/修改 → 从 Skills Hub 安装 |
| 信息增量 | Skills 的本质不是「插件」而是「程序性记忆」— 告诉 Agent 怎么做而不是替它做 |
| 写作风格 | 概念 + 实操穿插 |
| 字数 | ~3500字 |

### 第四篇：⌨️ 日常篇
**《Hermes 高频 Slash 命令：一天省 2 小时的终端技巧》**

| 项目 | 内容 |
|------|------|
| 核心卖点 | 30+ Slash 命令，用对一半效率翻倍 |
| 实操内容 | `/retry` `/undo` `/compress` → `/goal` → `/background` → `/steer` |
| 信息增量 | 很多人不知道的 `/steer` 注入式指令、`/branch` 分支会话 |
| 写作风格 | 清单式，分组讲解 |
| 字数 | ~3000字 |

### 第五篇：⏰ 自动化篇
**《让 Hermes 24 小时自动干活：Cron + Webhook 实战》**

| 项目 | 内容 |
|------|------|

| 信息增量 | Cron Prompt 设计原则（700字限制、Skill 拆分、One-shot） |
| 写作风格 | 场景驱动 |
| 字数 | ~3500字 |

### 第六篇：👥 多 Agent 篇
**《Hermes 蜂群模式：多个 Agent 并行协作开发大型项目》**

| 信息增量 | Advisor / Orchestrator 两种多智能体模式 |
| 写作风格 | 概念 + 实战演示 |
| 字数 | ~4000字 |

### 第七篇：🌐 网关篇
**《把 Hermes 接到微信/Telegram/钉钉：Gateway 配置全攻略》**

| 字数 | ~3500字 |

### 第八篇：🚀 实战篇
**《用 Hermes 从 0 到 1 发布一个完整项目》**

## 写作质量检查清单（每篇必过）

### 内容层
- [ ] 事实准确？（数据、时间、产品名称经过验证）
- [ ] 逻辑无矛盾？
- [ ] 结构合理，不跑题？
- [ ] 无编造信息？
- [ ] 每篇独立可读（不依赖读者看过上一篇）

### 风格层（花叔风格）
- [ ] 开头直接切主题，不绕圈子
- [ ] 短句为主（15-25字），口语化
- [ ] 每段有具体数字和事实
- [ ] 融入真实经历和案例
- [ ] 纯新手友好：解释每个专业术语的首次出现
- [ ] 不堆AI套话（在当今时代、综上所述、值得注意的是）
- [ ] 不假设读者有技术背景

### 细节层
- [ ] 句长不超过 30 字
- [ ] 段落手机 3-5 行
- [ ] 标点自然，多用句号
- [ ] 小标题每 300-500 字一个
- [ ] 加粗每 200-300 字 1-2 处
